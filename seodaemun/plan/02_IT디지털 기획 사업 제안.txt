# IT/디지털 기획 사업 제안 (필수 포함 항목)

=================================================
[IT-1] 서대문구 사업예산 투명공개 플랫폼 구축
=================================================
우선순위: 최우선
추정예산: 350~480백만원(1회성 구축비 250~330 + 연 운영비 50~75 × 2년)
제안부서(추정): 기획예산과 + 스마트정보과(구축), 전 부서(예산 데이터 제공)

[문제 정의]
spec/07 조사 결과, 2026년 서대문구 5대 분야 24개 사업 중 예산액이 구체적으로
확인된 사업은 단 2건(행복한 밥상 13.6억원, 주민참여예산 11.5억원)뿐이며 나머지
22건(약 92%)은 "확인안됨"으로 분류됨. 원인으로 확인된 것은:
1. 2026년도 예산안(약 8,253억원) 보도자료가 "총액 + 전년대비 증가율" 위주라
사업별 배분액이 대부분 공개되지 않음(예외적으로 어르신 급식만 증액 스토리로
구체적 금액이 보도됨)
2. 민선8기 4개년계획 문서(sdm.go.kr/chief/M8_Manifesto/m8_02.do)는 실재하나
본 조사에서 3회 조회 시 인코딩 오류로 세부 정책분야명·과제 목록의 신뢰도가
낮았음(spec/01) - 이는 구민이 동일 문서에 접근해도 겪을 수 있는 문제일 가능성이
있음(추정)
3. 정비사업(가재울·홍제 등)은 민간 시행 성격이 강해 애초에 구 예산 편성 대상이
아닌 경우가 많아, "예산이 없다"와 "예산이 비공개다"가 혼재되어 있어 구민이
구분하기 어려움

[제안 내용]
1. 서울시 열린재정 포털 API를 서대문구 홈페이지에 임베드하고, 부서명을 몰라도
사업명/키워드로 바로 검색되도록 기본 검색 UI를 구청 자체 페이지에 재구성
2. 5대 분야(도시재생주거환경/경제일자리청년상권/복지건강/문화관광환경/안전
스마트행정) 필터 및 사업별 상세페이지(사업명, 담당부서, 예산현액, 집행률,
진행상태, "구 예산 편성 대상 여부" 플래그) 표준 양식 마련 - 특히 "구 예산
편성 대상 아님(민간사업)" 표시를 명확히 해 spec/02에서 확인된 "예산 확인안됨"과
"애초에 구 예산이 아님"을 구분
3. 민선8기 4개년계획 자료를 원본 PDF/HWP 대신 구조화된 텍스트/표 데이터로
재입력(부서 협조 필요)하여 인코딩 문제 없이 공개
4. 2026년도 예산안 보도자료에 나열된 사업명(예: "행복한 밥상")을 실제 예산과목과
1:1 매핑하는 마스터 코드 부여

[기술 스택 권장]
- 백엔드: 서울시 열린재정 포털 API 연동 우선(신규 개발 최소화)
- 프론트엔드: 반응형 웹, 검색엔진 색인 가능하도록 서버사이드 렌더링 적용(IT-3와
동일한 렌더링 원칙 공유)
- 예산 마스터데이터는 최소한 신규 발간분부터는 원본 텍스트/한글 파일 병행 제공을
관련 부서에 요청

[기대효과]
- 24개 사업 중 예산 확인율 8%(2건) -> 목표 60% 이상으로 개선
- "얼마가 어디에 쓰였는지" 구민 문의에 단일 창구로 응답 가능
- 향후 민선9기 4개년계획 수립 시에도 예산-사업 매핑 마스터데이터를 그대로 활용 가능

[IT-2] 정비사업 구역경계·진행상황 통합 안내 시스템
=================================================
우선순위: 최우선
추정예산: 220~320백만원
제안부서(추정): 도시재생과 + 주택정비과

[문제 정의]
spec/02에서 실제로 확인된 문제: 홍제역세권활성화/유진상가·인왕시장 재개발(49층,
1,121세대+오피스텔92실)과 홍제역세권 장기전세주택 사업(38층, 3,026세대)이 서로
다른 수치로 보도되어, 본 조사에서도 두 사업의 동일성을 확정하지 못했음(1-3, 1-4).
여기에 더해 가재울뉴타운 7·8구역, 홍제4구역, 홍은13·15구역까지 최소 6개 이상의
정비사업이 동시 진행 중이나 이를 종합 안내하는 창구는 확인되지 않음. 조사자(본
리서치)조차 여러 출처를 대조해도 사업 경계를 확정하지 못했다는 것은, 정비구역
경계에 걸쳐 있거나 인접한 주민이 "내가 어느 사업의 대상인지" 헷갈릴 실질적
위험이 있음을 시사함.

[제안 내용]
1. 가재울뉴타운(7·8구역), 홍제역세권활성화(유진상가·인왕시장), 홍제4구역, 홍은
13·15구역 등 서대문구 내 모든 정비사업을 지도 기반으로 시각화하고, 각 사업의
공식 사업명·법정 구역번호·시행단계(정비구역지정/사업시행인가/관리처분/착공 등)를
표준화된 값으로 표시하는 "서대문 정비사업 진행현황 지도" 구축
2. 특히 홍제동 일대처럼 여러 사업이 인접·중첩되어 보도되는 지역은 "이 사업은
OO사업과 다른 사업입니다/같은 사업의 OO단계입니다"를 명시하는 사업 식별 안내
페이지를 우선 구축(spec/02의 1-3·1-4 혼선을 실제 해소하는 첫 사례로 활용)
3. 정비구역 내 소상공인(유진상가·인왕시장 기존 상인 등)을 위한 별도 안내 섹션
마련 - FN-1과 연동
4. IT-1(예산공개플랫폼)과 연동해 정비사업 관련 구 예산 집행분(있는 경우)을
표시

[기대효과]
- 정비사업 관련 정보 비대칭·경계 혼선 해소
- 최소 6개 이상 동시 진행 중인 대규모 정비사업(수천 세대 영향권)에 대한 주민
신뢰도 제고

[IT-3] 민원처리 현황 공공데이터 개방 및 게시판 렌더링 개선
=================================================
우선순위: 상
추정예산: 280~380백만원(플랫폼 재구축 200~260 + 연 운영비 40~60 × 2년)
제안부서(추정): 스마트정보과 + 총무과(민원 소관)

[문제 정의]
spec/08에서 본 조사가 직접 확인한 문제: "구청장에게 바란다" 게시판을 동일 URL로
3회 조회했을 때, 게시글 총수(21,913건)는 일관되게 나왔으나 최상단 게시글의
제목이 매번("민원과 건축허가 관련 질의" -> "도시철도 공사현장 주변 소음 민원" ->
"구청인사과 관리직 신규 공개채용 모집 공고..")으로 전혀 다르게 나왔음. 이는
해당 게시판이 JavaScript 클라이언트 렌더링 방식이라 외부 도구(검색엔진 크롤러
포함)가 실제 게시글 내용을 읽지 못하는 구조일 가능성을 시사한다. 이 문제가
사실이라면:
1. 구민이 네이버/구글 등에서 "서대문구 민원 OOO"으로 검색해도 본인의 민원이나
유사 민원 사례가 노출되지 않을 가능성
2. 언론·연구기관 등 외부에서 서대문구 민원 통계를 분석하려 해도 접근이 어려움
3. 본 조사(spec/00~08)처럼 구정 조사 자체가 왜곡될 위험(실제로 이번 조사에서
발생함)

[제안 내용]
1. 게시판을 서버사이드 렌더링(SSR) 또는 정적 페이지 생성 방식으로 전환해, 검색
엔진과 외부 조회 도구가 실제 게시글 목록을 읽을 수 있도록 개선
2. 민원 처리현황(월별/부서별 접수건수, 평균 처리일수 등 통계, 개인정보 제외)을
공공데이터포털 형식의 개방 API로 별도 제공 - 개별 게시글 원문 공개가 아니라
"통계 데이터"부터 우선 개방해 개인정보 이슈 최소화
3. "내 민원 조회" 페이지: 접수번호 기반으로 진행상황 확인 가능(접수->배정->
검토중->답변완료 공통 상태값)
4. 서대문구 빅데이터 페이지(sdm.go.kr/sdmbigdata)와 연동해 민원 통계를 기존
공개 통계 체계에 편입

[기대효과]
- 민원 데이터의 외부 신뢰성 있는 접근 경로 확보(이번 조사 같은 hallucination
기반 오정보 생산 자체를 방지)
- 구민의 민원 처리 투명성 체감도 제고
- 향후 IT-1과 연동해 "이 민원이 어느 예산 사업과 관련있는지"까지 연결 가능

[IT-4] 민선9기 공약이행 추적 플랫폼
=================================================
우선순위: 상
추정예산: 130~200백만원
제안부서(추정): 기획예산과 + 홍보과

[문제 정의]
spec/06: 신임 박운기 구청장(민선9기, 2026.7.1 취임)의 공약 중 서대문형 ESG,
AI 기반 복지, 안전통학로 3건이 명칭만 확인되고 세부 계획은 "확인안됨"으로
분류됨(취임 약 3주 시점). spec/03의 AI청년특구 역시 후보 시절 공약 수준에
머물러 취임 후 정책화 여부가 불확실함. 이는 취임 초기라 공약이 아직 예산·
세부계획으로 구체화되지 않은 자연스러운 단계이나, 역으로 지금이 투명성 설계를
반영할 수 있는 시점이기도 함.

[제안 내용]
1. 박운기 구청장의 전체 공약(주민자치 부활, 유진상가·인왕시장 개발, 강북횡단선·
서부경전철, 청년 주거·문화 지원, AI기반복지, 서대문형ESG 등)을 이행 단계(공약확정
-> 실천계획수립 -> 예산반영 -> 착수 -> 완료)로 추적하는 페이지를 사전 설계
2. IT-1과 동일한 코드 체계로 예산이 반영되는 순간부터 자동 연동
3. 1호 결재 사항인 "주민자치회 활성화"(14개동 목표)를 1호 추적 대상으로 우선
등록해 초기 신뢰도를 확보
4. 취임 후 100일 전후(통상적 공약 확정 관행으로 추정, 서대문구 자체 확인 사실
아님 - 명확히 단서 표기 필요)에 맞춰 1차 공개 목표로 로드맵 수립

[기대효과]
- 신규 행정부 출범 초기부터 투명성을 내장해 "선거 공약과 실제 집행"의 괴리
문제 예방
- 취임 초기 관심도가 높은 시점을 활용해 구민 참여도 제고
