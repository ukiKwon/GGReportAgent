# 실행 로드맵 및 기대효과

=================================================
1. 연차별 실행 로드맵 (2026년 하반기 제안 기준)
=================================================

[2026년 하반기 - 착수/기획]
- IT-4 민선9기 공약이행 추적 플랫폼: 신임 구청장의 공약 100일 확정 시점(취임 후
약 100일, 2026년 10월경 예상 - 확정된 사실 아님, 일반 관행 참고 추정) 전에 표준
틀 사전 설계 착수 - 가장 시급한 "타이밍성" 과제. 1호 결재사항인 주민자치회
활성화(spec/06, 5-1)를 1호 추적 대상으로 우선 등록
- IT-2 정비사업 통합 안내 시스템: 홍제역세권활성화/유진상가·인왕시장과 홍제역세권
장기전세주택 사업 간의 수치 불일치(spec/02, 1-3·1-4)부터 우선 확인·정리하는
실태조사 착수 - 이는 도시재생과 내부 확인만으로도 즉시 가능한 저비용 작업
- IT-1 예산공개플랫폼: 서울시 열린재정 포털 API 연동 가능성 검토
- IT-3 민원 데이터 개방: 게시판 렌더링 방식(JS 클라이언트 렌더링 여부) 현황
점검부터 착수 - 본 조사에서 확인된 hallucination 위험(spec/08)의 원인이 실제
렌더링 방식 때문인지 내부 확인 필요
- FN-1 정비사업 이주지원금: 유진상가·인왕시장 등 우선 대상구역 실태조사

[2027년 - 구축/1차 시행]
- IT-4: 민선9기 공약 확정안이 나오는 즉시 공약이행 추적 페이지 1차 오픈
- IT-2: 정비사업 진행현황 지도 베타 오픈(가재울·홍제 우선)
- IT-1: 예산공개플랫폼 베타 오픈, 24개 사업 중 우선 확인 가능했던 사업부터 순차
반영
- IT-3: 민원 통계 개방 API 1차 구축(개인정보 제외 통계부터)
- FN-1, FN-2: 지원금 신청 접수 개시(FN-2는 3호점 위치 확정 후)

[2028년 - 확산/고도화]
- IT-1: 위원회/부서 예산자료의 전면 구조화 완료, 전체 24개+ 사업 예산 매핑 완성
- IT-2: 서대문구 내 전체 정비구역(홍제4구역, 홍은13·15구역 등 포함) 진행현황
지도 정식 오픈
- IT-3: 게시판 서버사이드 렌더링 전환 완료, "내 민원 조회" 페이지 정식 운영
- FN-3: 청년 창업 초기정착 지원금 1기 운영(AI청년특구 정책화 여부에 따라 규모
조정)

[2029년 - 정착/성과평가]
- IT-1~IT-4 전체 서비스 안정화, 유지보수 체계 전환
- FN-1~FN-3 성과평가 및 민선9기 차기 4개년계획(수립 시) 반영 검토
- 정비사업 진행 단계에 맞춰 FN-1 한시사업 단계적 조정

2. 핵심성과지표(KPI) 제안
=================================================

[IT-1 예산공개플랫폼]
- 24개 사업 중 예산 확인 가능 비율: 현재 8%(2/24) -> 목표 60% 이상
- 월간 이용 세션 수, 사업별 페이지 체류시간

[IT-2 정비사업 통합 안내 시스템]
- 1-3·1-4 유형의 사업 경계 혼선 재발 건수(목표: 0건)
- 지도 서비스 이용자 수, 구역별 페이지 조회수

[IT-3 민원데이터 개방]
- 게시판 콘텐츠의 검색엔진 색인 여부(현재 미확인/의심 -> 목표: 정상 색인 확인)
- 통계 API 호출 건수, "내 민원 조회" 이용률

[IT-4 공약이행 추적 플랫폼]
- 민선9기 공약 확정 후 플랫폼 반영까지 소요기간
- 공약별 이행률 공개 항목 수

[FN-1 정비사업 이주지원금]
- 지원 신청 건수 및 집행률(구역별)
- 정비구역 관련 민원 건수 추이(단, spec/08 접근불가 문제가 해소되어야 측정 가능
- IT-3 선행 필요)

[FN-2 행복한 밥상 확대]
- 3호점 개소 여부, 이동식 배달 서비스 이용 어르신 수
- 14개 행정동 중 서비스 권역 커버 비율

[FN-3 청년창업 지원금]
- 지원 청년 수, 창업 유지율(1년 후 폐업률 등)
- AI청년특구 정책화 여부에 따른 후속 확장 검토

3. 리스크 및 대응
=================================================

[리스크 1] IT-2 착수 전제인 "1-3·1-4 사업 동일성 확인"이 실제로는 완전히 다른
2개 사업일 경우, 시스템 설계 자체를 재검토해야 함
- 대응: 2026년 하반기 실태조사를 로드맵 최우선 과제로 배치해 이 불확실성을
가장 먼저 해소

[리스크 2] IT-3가 전제하는 "게시판이 JS 렌더링이라 문제"라는 진단이 틀렸을 경우
(예: 실제로는 정적이나 WebFetch 도구 자체의 한계였을 경우), 불필요한 재구축
비용이 발생할 수 있음
- 대응: 로드맵 최우선 단계에서 "현황 점검"을 먼저 수행하고, 실제 원인이 확인된
후에만 본 구축 착수(사전 확정 설계 금지)

[리스크 3] FN-3가 전제하는 AI청년특구가 실제로 정책화되지 않을 경우, 예산 취지가
불명확해질 수 있음
- 대응: FN-3 예산을 AI청년특구 여부와 무관하게 집행 가능한 소규모 지원(로컬
크리에이터 연계)으로 한정해 설계, 특구 정책화 시 별도 확장 검토

[리스크 4] 서대문구 재정자립도(20.3%)가 종로구(42.0%)보다 낮아 자체 재원만으로는
전체 사업 추진이 버거울 수 있음
- 대응: 국비/시비 공모사업(특히 정비사업 관련 국토부·서울시 사업, 청년정책
관련 서울시 사업) 연계를 전제로 예산 규모를 종로구 대비 낮게 설계함(0.20~0.29%)

4. 종합 기대효과
=================================================
1. 예산 투명성: 24개 사업 중 22개(92%)가 예산 확인 불가능했던 구조적 문제(spec/07)
를 해소해 정책 신뢰도 제고
2. 정비사업 정보격차 해소: 홍제동 일대 복수 사업 간 경계 혼선(spec/02) 사례를
해소하고, 최소 6개 구역·수천 세대 영향권의 대규모 정비사업에 대한 주민 수용성
제고
3. 데이터 신뢰성 회복: 본 조사에서 실제로 발견된 민원게시판 hallucination
위험(spec/08)을 근본적으로 해소해, 향후 유사 조사·구민 스스로의 정보 접근 모두
개선
4. 행정 전환기의 기회 활용: 민선9기 출범 초기(공약 미확정 단계)라는 드문 시점을
활용해 투명성·추적성을 처음부터 내장한 시스템 설계
5. 예산 효율성: 신규 대규모 시스템 구축보다 기존 인프라(서울시 열린재정 포털,
서대문구 빅데이터 페이지, 열린청장실) 재활용 중심으로 설계해 총 사업비를 서대문구
2026년 예산(약 8,253억원) 대비 약 0.2~0.3% 내외로 억제
6. 취약계층 보호: 정비사업 소규모 소유자·세입자, 거동불편 어르신, 청년 창업가 등
정보·자금력이 상대적으로 열세인 집단에 대한 금전적 지원을 확인된 실수요 기반으로
설계(단, FN-1·FN-3는 실제 민원 사례가 아닌 구조적 위험 추정에 기반한 선제적
제안임을 명시)
