IT/디지털 기획 사업 제안 (필수 포함 항목)

=================================================
[IT-1] 노원구 통합 사업추진현황 검색 포털 구축
=================================================
우선순위: 최우선
추정예산: 400~550백만원 (1회성 구축비 280~380 + 연 운영비 60~90 × 2년)
제안부서(추정): 정보통신과 + 기획예산과 (구축), 전 부서(콘텐츠 제공)

[문제 정의]
spec/08 조사 결과, 23개 표본 사업 중 확인됨은 65%(15개)이나, 이 중 다수가 nowon.kr 자체
페이지가 아니라 council.nowon.kr(구의회 회의록), 언론보도, cleanup.seoul.go.kr(서울시
정비사업 정보몽땅) 등 "구청 홈페이지 밖"에서 확인되었다는 점이 핵심 문제. spec/08 패턴분석
원문: "nowon.kr 자체 페이지보다 council.nowon.kr(구의회 회의록)과 외부 언론/서울시 보도자료를
통해 확인되는 사업이 훨씬 많음. 노원구 홈페이지는 사업개요·정적 소개 위주이고, 실질적 추진현황은
의회 회의록 쪽에 더 풍부하게 축적되어 있음(도봉구의 majorWork DB와 유사한 역할을 회의록
시스템이 대신함)". 즉 주민이 "우리 동네 사업이 지금 어디까지 진행됐나"를 알려면 구청
홈페이지가 아니라 의회 회의록을 검색해야 하는 역전 현상이 존재.

[제안 내용]
1. council.nowon.kr(회의록), data.nowon.go.kr(열린데이터광장), 각 부서 사업설명 페이지를
   백엔드에서 정기 수집(회의록 키워드 매칭 방식)해, 2026년 주요업무계획 6개장 약 260개 사업을
   표준 메타데이터(사업명, 장/소주제 코드 예: "1장-동북권新경제중심지조성", 담당부서, 예산,
   추진현황, 관련 회의록/보도 링크)로 정리한 마스터 DB 구축
2. 검색 UX: 사업명/키워드/분야(6대 구정목표)/담당부서/추진상태(계획중·추진중·완료) 필터 제공,
   특히 spec/08에서 "확인안됨"으로 분류된 새활용센터·진로직업체험센터·재건축진단비용지원과
   같이 예산은 배정되었으나 전용 페이지가 없는 사업을 우선 콘텐츠화
3. 각 사업 페이지에 회의록 발췌·언론보도 링크·서울시 정비사업 정보시스템 링크를 함께 노출해
   "구청 홈페이지 한 곳에서 회의록까지 확인 가능"하도록 통합
4. 신속추진단(spec/02, 재건축·재개발) 등 주민 관심이 높은 대형사업은 별도 진행률 게이지 UI로
   강조 표시

[기술 스택 권장]
- 검색엔진: 경량 오픈소스 검색엔진(Meilisearch 등), 저비용
- 프론트엔드: 반응형 웹, SSR 적용으로 SEO 대응(현재 nowon.kr 콘텐츠가 검색엔진에 "확인안됨"
  판정을 받는 문제 개선)
- 백엔드: council.nowon.kr 회의록 정기 크롤링/파싱 배치, data.nowon.go.kr API 연동

[기대효과]
- 회의록·언론보도 의존 구조를 구청 홈페이지 중심으로 흡수해 정보 접근성 개선
- spec/08 확인안됨 3개(새활용센터, 진로직업체험센터, 재건축진단비용지원) 우선 해소
- 연간 계획 갱신 체계(spec/01, 노원구는 4개년계획이 아닌 연간 주요업무계획 운영)에 맞춰
  매년 마스터DB 갱신만으로 유지보수 가능

[IT-2] 민원 채널 통합 대시보드 & 알림톡 연동
=================================================
우선순위: 최우선
추정예산: 300~450백만원 (구축비 250~350 + 연 알림톡 발송비 50~100)
제안부서(추정): 감사담당관(고충처리위원회 소관) + 정보통신과

[문제 정의]
spec/09 조사 결과:
1. 실사용 민원/제안 게시판이 최소 5개(구청장에게바란다, 칭찬합니다, 오늘의제안, 주민참여예산,
   구민고충처리위원회)로 나뉘어 있고, 7장 문서수치 기준 2025년 총 민원접수는 124,950건
   (구청장에게바란다 1,088 + 새올상담민원 44,077 + 응답소 78,705 + 182민원 1,080)으로
   집계 시스템 자체가 최소 4개로 분산.
2. "구청장에게 바란다"는 누적 접수번호가 20,032,300번대(2026-07-16 기준)에 달할 만큼 규모가
   크고 목록상 "처리 중" 상태가 다수이며, 다수 게시글이 "비공개글"로 마스킹되어 상세내용 확인
   불가 - 처리 경과 추적이 어려움.
3. 반면 "구민고충처리위원회"는 "담당자지정 -> 답변완료" 흐름이 명확해 상대적으로 신속
   (예: 상수도 공사 흔들림 민원 2026-07-04 답변완료, 도로반사경 교체 요청 2026-06-24 답변완료,
   해충 방제요청 2026-06-21 담당자지정). "오늘의 제안"도 "스마트노원핏 인센티브 개선"
   (2026-06-29 답변완료) 등 완료 표시가 명확 - 채널 간 처리 투명성 격차가 spec/09에서 실측됨.
4. "협치노원(협치 와글와글)"은 형식상 게시판으로 남아있으나 2023~2024년 이후 게시글 없이
   사실상 휴면 상태(spec/09) - 운영 채널 목록에 죽은 채널이 그대로 노출되어 주민 혼란 유발 가능.

[제안 내용]
1. 5개 활성 민원/제안 채널(구청장에게바란다/칭찬합니다/오늘의제안/주민참여예산/구민고충처리위원회)
   + 외부연계 시스템(새올상담, 응답소, 182민원)의 접수번호를 통합 관리하는 백엔드 대시보드 구축
2. 주민용 "내 민원 조회" 페이지: 접수번호+휴대폰 인증으로 채널 무관하게 진행상황 실시간 확인
   (접수 -> 담당부서 배정 -> 검토중 -> 답변완료 공통 상태값 정의, 구민고충처리위원회 수준의
   투명성을 전 채널로 확산)
3. 카카오 알림톡 연동: 상태 변경 시 자동 알림 발송 - "구청장에게 바란다"처럼 목록에서 진행상황이
   드러나지 않는 채널에 우선 적용
4. 휴면 채널("협치노원") 정리: 신규 게시 중단 안내 또는 활성 채널로 통합, 홈페이지 메뉴에서
   혼선을 줄임
5. 연 1회 "민원처리 통계 공표"(구민고충처리위원회는 spec/09에서 2025년 처리 78건 상세내역
   -시정권고1·의견표명13·안내회신20 등- 이 이미 공개되고 있음, 이 형식을 전 채널로 확대)

[기술 스택 권장]
- 기존 시스템(nowon.kr/mayor/, eminwon.nowon.kr 새올전자민원) API/DB 연동 어댑터 개발
- 알림톡: 카카오 비즈메시지 API
- 대시보드: 저코드(Low-code) 툴 또는 기존 새올전자민원 인프라 확장 검토

[기대효과]
- 구민고충처리위원회 수준(답변완료 흐름 명확)의 투명성을 "구청장에게 바란다" 등 대형 채널로 확산
- 채널 파편화 해소로 향후 민원 데이터 기반 정책 수립(어느 분야·지역에 민원이 집중되는지) 가능
- 124,950건(spec/09) 규모의 민원 데이터를 통합 관리해 중복 접수·반복 민원 감소 기대

[IT-3] 열린데이터광장-홈페이지 연동 및 서브도메인 통합검색
=================================================
우선순위: 상
추정예산: 150~220백만원
제안부서(추정): 정보통신과

[문제 정의]
spec/08: 노원구는 기능별로 최소 9개 서브도메인(council-의회, ecocenter-환경, gongdan-공단,
webzine-뉴스레터, law-법무, land-부동산, data.nowon.go.kr-열린데이터광장, eminwon-전자민원,
nowon.kr/mayor-열린구청장실)에 기능이 분산되어 있음. 특히 열린데이터광장은 항목14에서
"부분확인"으로 분류 - "nowon.kr 내 열린행정>정보공개>공공데이터개방 메뉴 존재확인. 실제
데이터포털은 별도 서브도메인 data.nowon.go.kr(nowon.kr이 아닌 nowon.go.kr 도메인)로 확인"됨.
즉 본 도메인(.kr)과 데이터광장 도메인(.go.kr)이 달라 주민이 데이터광장의 존재 자체를
찾기 어려운 구조적 문제.

[제안 내용]
1. nowon.kr 메인/서브 메뉴에 9개 서브도메인으로 연결되는 통합 사이트맵 및 검색창(사이트 간
   교차 검색) 신설 - 별도 시스템 재구축 없이 기존 사이트들에 대한 "찾아가는 관문" 역할
2. data.nowon.go.kr(열린데이터광장) API를 nowon.kr 메인 페이지에 위젯 형태로 노출해
   예산 집행률, 사업 통계 등 정량 데이터를 주요업무계획 사업 페이지(IT-1)에서 자동 표시
3. council.nowon.kr(회의록) 검색 결과를 IT-1 통합포털에 자동 색인해 "확인안됨" 사업의
   회의록 언급 여부를 즉시 확인 가능하도록 연동

[기대효과]
- 열린데이터광장 등 유용하지만 묻혀있는 서브도메인의 발견 가능성 제고
- IT-1과 결합해 스텔스 상태였던 9개 서브도메인 콘텐츠의 실질적 활용도 증가

[IT-4] 주민참여예산-계획사업 연계 매핑 서비스
=================================================
우선순위: 중
추정예산: 80~120백만원
제안부서(추정): 기획예산과 + 홍보담당관

[문제 정의]
spec/09: "주민참여예산(주민제안사업)" 게시판에 2026-07-02 "석계로 동서 불균형 해소 및
보행안전" 제안이 접수되어 있음. 이는 spec/03(2장 교통도시)의 "석계역 일대 간판개선사업"
(659백만원, 2026.1~12, 상가 및 골목시장 177개소 39개동 대상)과 지리적으로 정확히 겹치는
민생 이슈이나, 주민참여예산 제안과 기존 계획사업이 서로 다른 트랙에서 별도로 관리되어
주민 입장에서는 "내가 낸 제안이 기존 계획과 어떻게 연결되는지" 알 수 없는 구조.
2026-07-03 "노원 본점 소상공인 IP 로컬 브랜딩 지원사업" 제안도 spec/07의 "찾아가는
소상공인 매니저 운영"(75백만원) 등 기존 소상공인 지원사업과 연계 가능성이 있으나 매핑되어 있지 않음.

[제안 내용]
1. IT-1 마스터DB와 연동해, 주민참여예산에 신규 제안이 접수될 때 기존 계획사업(6개장 약 260개)
   중 유사도가 높은 사업을 자동 매칭해 제안자와 담당부서에 안내(완전 자동판정이 아닌
   담당자 검토용 추천 리스트 제공)
2. 지역(동)/키워드 기반으로 "이 지역 관련 기존 계획 사업" 지도를 주민참여예산 제안 화면에
   노출해 중복 제안을 줄이고 기존 사업 인지도를 높임
3. 채택된 제안이 실제 계획에 반영되는 과정을 IT-1 포털에서 추적 가능하도록 표시

[기대효과]
- 계획-제안 간 단절 해소로 주민 체감 정책 설계력 강화
- 향후 예산 심의 시 "주민 제안과 실제 계획의 연결 사례"를 성과 지표로 활용 가능
