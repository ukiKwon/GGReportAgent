[IT-1] 신설사업 전용페이지 패스트트랙 & 통합사업정보 검색 플랫폼
=================================================
우선순위: 최우선
추정예산: 350~500백만원 (1회성 구축비 250~350 + 연 운영비 50~75 × 2년)
제안부서(추정): 정보통신과 + 홍보담당관 (구축), 전 부서(콘텐츠 제공)

[문제 정의]
spec/08 조사 결과, 44개 표본사업 중 확인됨 약 70%(31개), 부분확인 약 25%(11개),
확인안됨 약 5%(2개)로 전체 확인율 자체는 낮지 않으나, "부분확인" 11개 중 다수가
공교롭게도 2026년 신규 개관·전환 사업에 집중됨:
- 광진 청년아지트(2026.6 개관): 청년포털 새소식·리플렛만 확인, 위치·시설 전용페이지 없음
- 광진 천천히 편의점(2026.3 개관): 단신 확인, 상세 위치·운영기관 소개페이지 없음
- 화양생활지원센터: 채용공고에서 명칭만 확인, 운영시간·서비스 상세페이지 없음
- 중곡지역 소규모 노인복지센터: 민선8기 공약지도에 항목만 존재, 추진현황 페이지 없음
spec/08은 이를 "신설 초기 콘텐츠 축적 시차"로 분석함 - 즉 예산이 있고 사업이 실제
개관/운영되어도 "막 시작한 시점"에는 검색 사각지대가 구조적으로 발생함.

[제안 내용]
1. "신설사업 전용페이지 패스트트랙" 프로세스 도입: 사업 착수(개관 3개월 전) 시점에
   담당부서가 표준 템플릿(위치/기간/대상/이용방법/문의처)으로 최소 정보를 등록하고,
   정식 상세페이지 완성 전까지 임시 안내페이지를 우선 노출하는 규정을 신설
2. 기존 청년포털·통합돌봄 전용페이지(menuNo=201860)·지구단위계획 메뉴(menuNo=200543~)
   등 광진구가 이미 체계적으로 운영 중인 분야별 메뉴 구조(spec/08 패턴분석 1번)를
   표준으로 삼아, 아직 이 체계에 편입되지 않은 신설사업을 우선 편입
3. 전체 사업 통합검색 페이지 신설: 6대 분야(균형발전/상생복지/경제활력/문화교육/
   안전환경/열린소통) 및 추진상태(계획중/개관예정/운영중) 필터 제공, 2026년 주요업무계획
   PDF상 사업명을 그대로 검색 키워드로 등록해 정책문서와 홈페이지 간 대조 검색 지원
4. 열린데이터광장 등 서울시 공통 데이터 인프라 연동 가능성 우선 검토(신규 개발 최소화)

[기술 스택 권장]
- 검색엔진: 경량 오픈소스(Meilisearch 등)
- 프론트엔드: 기존 gwangjin.go.kr 포털 프레임워크 내 신규 메뉴로 추가(별도 사이트
  분리 지양)
- 백엔드: 각 부서 콘텐츠관리시스템(CMS) 입력 폼에 "패스트트랙" 템플릿 필드 추가

[기대효과]
- 신설사업 부분확인 비율(약 25%) 축소, 특히 개관 초기 3~6개월 구간의 정보공백 해소
- 청년아지트·천천히편의점 등 이미 예산·인력이 투입된 사업의 조기 체감도 확보
- 향후 신설되는 모든 사업에 동일 프로세스 적용 가능해 재발 방지

[IT-2] 민원채널 처리상태 표준화 & 통합 조회 시스템
=================================================
우선순위: 최우선
추정예산: 300~450백만원 (구축비 250~350 + 연 알림 발송비 50~100)
제안부서(추정): 감사담당관(옴부즈만 소관) + 정보통신과

[문제 정의]
spec/09 조사 결과, 실제 게시판이 최소 7종(경호와의 소통방/온라인정책방/주민참여예산
운영개선의견/칭찬합시다/설문조사/온라인접수/옴부즈만 민원신청)으로 분산되어 있고,
채널마다 처리상태 표기 체계 자체가 다름:
- "경호와의 소통방": "답변완료"/"행정참고" 2종만 존재, "접수"/"처리중" 표기가 없음
- "온라인정책방": 접수준비/처리중/답변완료가 고르게 분포
- "주민참여예산-운영개선의견": 접수준비 상태가 다수
- "옴부즈만 민원 신청": 목록엔 제목·등록일만 공개, 처리상태는 로그인 후에만 확인 가능
  (도봉구 옴부즈만 게시판보다도 공개범위가 좁음 - spec/09 4번)
- "주민참여예산 사업제안" 게시판은 안내문만 노출, 실제 제출 게시글 목록 자체가 없음
  (spec/09 3-1)
같은 민원이라도 채널에 따라 상태값 체계가 달라 주민이 일관되게 진행상황을 추적할
방법이 없고, 참여형 제안(사업제안)은 제출 후 결과 확인 경로 자체가 불투명함.

[제안 내용]
1. 4단계 공통 상태값(접수 -> 담당부서 배정 -> 검토중 -> 답변완료)을 전 채널에 공통
   적용하는 표준을 수립하고, "경호와의 소통방"처럼 기존에 "행정참고"만 쓰던 채널에도
   이 표준을 소급 적용
2. 접수번호+휴대폰 인증 기반 "내 민원 조회" 통합 페이지 신설 - 경호와의 소통방/
   온라인정책방/옴부즈만/주민참여예산 4개 채널을 한 화면에서 조회
3. 주민참여예산 "사업제안" 게시판에 실제 제출 게시글 목록을 노출하도록 개선(현재
   안내문만 존재하는 구조 개선) - 제안 -> 심사 -> 선정여부까지 추적 가능하게 함
4. 카카오 알림톡 연동으로 상태 변경 시 자동 알림, 특히 상태표기가 빈약한 "경호와의
   소통방" 채널에 우선 적용
5. 연 1회 "민원처리 통계 공표"를 온라인정책방 등 기존 채널을 통해 자동 생성 -
   채널별 평균 처리일수, 유형별 건수 투명 공개

[기술 스택 권장]
- 기존 시스템(gwangjin.go.kr 각 게시판, 새올전자민원창구) API/DB 연동 어댑터 개발
- 알림톡: 카카오 비즈메시지 API
- 통합조회: 저코드 툴 또는 기존 포털 프레임워크 확장

[기대효과]
- 채널 간 상태표기 불일치 해소로 행정 신뢰도 제고
- "사업제안 후 결과를 알 수 없다"는 참여형 민원의 구조적 불투명성 해소
- 2026년 확인된 최다 민원 유형(재개발/정비사업, 교통·보행)의 처리현황을 주민이
  채널 구분 없이 확인 가능

[IT-3] 모아타운 등 정비사업 통합 진행상황 알림 플랫폼
=================================================
우선순위: 상
추정예산: 200~300백만원
제안부서(추정): 도시재생과 + 정보통신과

[문제 정의]
spec/02: 균형발전도시 26개 세부사업 중 모아타운만 9개 이상 구역(구의3동 2곳, 광장동,
자양1동 3곳, 자양2동 3곳)이 동시에 "관리계획 수립 중" 또는 "대상지 선정 후 수립 예정"
단계로 진행 중이며, 이 외에도 신속통합기획 재건축·재개발(자양3동, 자양4동A구역,
중곡1동 등)이 다수 동시 진행. spec/08: 이 중 자양3촉진구역(옛청사부지), 화양초
폐교부지처럼 구청 홈페이지 전용페이지가 없이 구의회 회의록에만 진행상황이 남는
사업이 다수 확인됨. spec/09: 실제로 "구의2동 재개발 반대", "모아주택 정비사업
비례율 개선"처럼 정비사업 관련 민원이 경호와의 소통방·온라인정책방 공통 상위
패턴으로 확인됨.

[제안 내용]
1. 26개 균형발전도시 사업(지구단위계획+모아타운+재건축·재개발) 전체를 지도 기반으로
   시각화하는 "광진 정비사업 현황지도" 페이지 신설 - 구역별 현재 단계(대상지 선정/
   관리계획 수립중/정비구역 지정/조합설립 등)를 한눈에 표시
2. 각 구역 페이지에 최신 추진현황(spec/02에서 이미 구체적으로 파악된 일자별 이력,
   예: 자양5촉진구역 재정비촉진계획 변경절차 완료 2026.2.26.)을 정기 갱신
3. 정비사업 관련 민원(IT-2 통합조회 시스템의 태그 기능 활용)을 구역별로 자동 집계해
   도시재생과·주거사업과가 주간 단위로 확인하는 프로세스 수립
4. 구역 지정 등 주요 단계 변경 시 해당 구역 인근 주민에게 알림(동 단위 안내문자 또는
   앱 알림) 발송 검토

[기대효과]
- 자양3촉진구역, 화양초 폐교부지처럼 구의회 회의록에만 남아있던 진행상황을 주민이
  직접 확인 가능
- 정비사업 관련 민원(2026년 확인된 상위 패턴)에 대한 선제적 정보 제공으로 반복 민원 감소
- 9개 이상 모아타운 구역이 동시 진행되는 광진구 특유 상황에서 행정 투명성 확보

[IT-4] 골목소통 등 현장활동형 사업 온라인 기록화
=================================================
우선순위: 중
추정예산: 60~100백만원
제안부서(추정): 홍보담당관 + 각 동주민센터

[문제 정의]
spec/08: 44개 표본사업 중 유일하게 "확인안됨"으로 분류된 사업이 "골목소통" -
청소·경로·돌봄·봉사 등 5개 분야로 실제 운영되고 있으나 별도 전용페이지나 온라인
콘텐츠화 없이 동 단위 현장활동으로만 수행되어 검색 자체가 되지 않음(spec/08 6장
요약, 패턴분석 6번). 반면 같은 열린소통도시 분야의 "주민합동 야간순찰"은 보도자료
(nttId=6352827)로 자양1동 사례가 구체적으로 확인되어 대조를 이룸.

[제안 내용]
1. 주민합동 야간순찰(spec/07: 2026년 3회 진행, 총 27건 지적사항 발굴)에 적용 중인
   "활동 후 보도자료/게시물 등록" 방식을 골목소통 5개 분야에도 동일 적용
2. 동주민센터 게시판에 "이달의 골목소통" 형태로 활동 사진과 요약을 반자동 템플릿으로
   등록하는 간단한 워크플로우 도입(완전 자동화보다 템플릿+검수 방식)
3. IT-1 통합검색 플랫폼과 연동해 골목소통 활동도 검색 가능하도록 등록

[기대효과]
- 저비용으로 유일한 "확인안됨" 사업의 사각지대 해소
- 현장 중심 소통행정(동지역책임제, 골목소통)의 실제 성과를 대외적으로 입증하는 근거자료 확보
