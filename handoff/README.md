# handoff/ 폴더 안내

**대시보드(report/5-report.html) subagent-driven-development 실행에 한정된** 세션 재개용 체크포인트입니다. 다른 작업 스레드(예: 25개 자치구 일괄생성 프로젝트)는 다루지 않습니다.

## 생성 규칙
- 태스크 3개가 완료될 때마다 `작업일자일시.md`(예: `20260719_1142.md`) 파일을 새로 생성합니다.
- 파일명에 주제를 넣지 않는 이유: "이 시점의 전체 상태"를 나타내는 스냅샷이지, 특정 이벤트 기록이 아니기 때문입니다.

## 고정 구조
각 파일은 아래 3단 구성을 따릅니다.
1. **완료된 태스크** — 이번 배치(최대 3개)에서 끝난 태스크 요약
2. **현재 상태** — 전체 진행률, 관련 태스크리스트/원장 경로
3. **다음 단계** — 다음에 할 태스크, 다음 handoff 트리거 조건

## 용도
세션이 압축되거나 재시작됐을 때, 다른 프로젝트 로그를 뒤지지 않고 가장 최근 파일 하나만 읽으면 "어디까지 했고 다음에 뭘 해야 하는지" 바로 파악하기 위한 용도입니다. 상세 사실 기록이 필요하면 [`data/`](../data/README.md) 폴더를 참고하세요.
