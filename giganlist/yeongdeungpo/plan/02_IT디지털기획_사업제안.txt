# IT/디지털 기획 사업 제안 (필수 포함 항목)

=================================================
[IT-1] 영등포구 사업예산-분야 매핑 공개 플랫폼 구축
=================================================
우선순위: 최우선
추정예산: 380~520백만원(1회성 구축비 260~360 + 연 운영비 60~80 × 2년)
제안부서(추정): 기획예산과 + 디지털행정과(구축), 전 부서(예산 데이터 제공)

[문제 정의]
spec/06 조사 결과, 2026년도 예산안 "희망예산"(4,020억원)·"행복예산"(328억원)의
세부 8개 항목(경제적약자 사회안전망 2,255억원 등)은 원문과 정확히 일치하는
수준까지 확인되나, 그 안에 구체적으로 어떤 사업명에 얼마가 배분되는지는 단
하나도 확인되지 않음(spec/04·05). 원인은:
1. 예산안 세부보도 자체가 파이낸스투데이 1개 매체의 단독 심층기사에 의존하고
있고(spec/06), 해당 기사에도 "분야 대분류 금액"까지만 나오고 개별 사업명·
사업별 배분액은 없음
2. 서울 열린재정 포털(openfinance.seoul.go.kr/ydp)의 "사업별 예산정보" 조회
페이지는 회계연도 2021~2026 선택 가능한 정식 조회 시스템처럼 보이나, 실제
접속 결과 "등록된 데이터가 없습니다"로 표시되어 전혀 조회되지 않음(spec/01
직접 확인 - 종로구 조사에서도 동일한 구조적 결함이 확인된 바 있어 자치구
개별 문제가 아니라 이 포털 자체의 공통 한계로 추정됨)
3. 홈페이지 투명행정>예산공개>중기지방재정계획 게시판에 2026~2030 계획 PDF는
존재하나(spec/01), 목록 페이지 수준에서는 제목·부서명만 확인되고 첨부파일
내부의 분야별 세부 투자계획표는 확인하지 못함

[제안 내용]
1. 구 자체적으로 예산안 보도자료의 3대 분야(희망/행복/미래예산) 8개 세부항목
각각에, 실제 집행되는 개별 "사업명" 목록을 매핑하는 마스터 코드 테이블을
구축(부서 협조 필요) - 예를 들어 "육아지원 정책 916억원" 항목 아래 실제
어린이집 지원, 출산장려금 등 몇 개 사업이 있고 각각 얼마인지 공개
2. 중기지방재정계획 PDF(이미 확보된 원본, spec/01)의 분야별 투자계획표를
구조화된 데이터로 재입력하여 홈페이지에 공개(첨부파일 PDF 대신 표/차트 형태)
3. 서울 열린재정 포털의 "등록된 데이터가 없습니다" 문제를 해소하기 위해 구
차원에서 서울시 재정담당 부서에 데이터 연동 상태를 문의하고, 임시로는 구
자체 홈페이지에 별도 예산-사업 매핑 페이지를 구축
4. 문래동4가(8,470억원)·여의도금융진흥계획(593.57억원)처럼 예산이 구체적으로
공개되는 대규모 사업과, 희망/행복예산처럼 대분류로만 공개되는 사업을 하나의
통합 페이지에서 함께 조회 가능하도록 표준 양식 마련

[기술 스택 권장]
- 백엔드: 서울 열린재정 포털 API 연동 시도 우선, 실패 시 구 자체 데이터베이스로
대체 구축
- 프론트엔드: 반응형 웹, 검색엔진 색인 가능하도록 서버사이드 렌더링

[기대효과]
- 대분류-개별사업 연결 부재 문제 해소, "얼마가 어디에 쓰였는지" 구민 문의에
단일 창구로 응답 가능
- 향후 중기지방재정계획(2026~2030) 개정 시에도 매핑 데이터를 그대로 활용 가능

[IT-2] 민원 처리현황 실시간 공개 & 옴부즈만 제도 정비
=================================================
우선순위: 최우선
추정예산: 250~350백만원(구축비 200~280 + 연 운영비 25~40)
제안부서(추정): 감사담당관 + 디지털행정과

[문제 정의]
spec/07 조사 결과:
1. "구청장에게 바란다" 게시판에서 확인된 10건 전부가 "처리중" 상태로, 언제
답변이 완료될지 구민이 예측할 방법이 없음(단, 이는 최신 게시글 위주로
조회되어 나타난 결과일 수 있고 뒤 페이지의 오래된 게시글은 확인하지 못한
조사 한계도 있음을 감안)
2. 감사담당관이 2건(86245 도로과 공동, 86239 단독)에 관여하고 있어 민원
접수·배정 총괄 역할을 하는 것으로 추정되나 명확한 업무분장은 확인안됨
3. 영등포구 자체 옴부즈만 제도의 존재 여부 자체가 검색으로 확인되지 않음 -
도봉구·종로구 사례(제도는 있으나 게시글만 확인안됨)보다 더 근본적인 정보
공백

[제안 내용]
1. "구청장에게 바란다" 접수 시점부터 "접수 -> 배정 -> 검토중 -> 답변완료"
공통 상태값을 표준화하고, 접수번호 기반으로 구민이 실시간 진행상황을 조회할
수 있는 페이지 구축(처리중인 민원의 평균 소요일수를 익명화된 통계로 공개)
2. 감사담당관의 민원 관련 역할(총괄 배정 여부 등)을 명문화하고, 이를 구민
안내페이지에 명시
3. 영등포구 자체 옴부즈만 제도 존재 여부를 먼저 감사담당관 차원에서 확인하고,
없다면 서울시 시민감사옴부즈만위원회와의 연계 창구를 명확히 안내하는 페이지
신설(제도가 있다면 "구청장에게 바란다"와 동일한 수준의 접근성 제공)
4. 연 1~2회 "민원처리 통계 공표"(부서별 접수건수, 평균 처리일수 등) 자동 생성

[기대효과]
- 전건 "처리중" 상태로만 보이는 현재의 불투명성 해소
- 옴부즈만 제도의 존재·부재를 명확히 하여 구민의 민원 채널 혼란 감소
- IT-1(예산공개플랫폼)과 연동해 "이 민원이 어느 예산 사업과 관련있는지"까지
연결 가능

[IT-3] 6개 정비구역 통합 안내 플랫폼(신길뉴타운·문래동)
=================================================
우선순위: 상
추정예산: 220~320백만원
제안부서(추정): 도시계획과 + 주택정책과

[문제 정의]
spec/03: 신길-영등포역 일대 6개 구역(신길2구역, 신길1구역, 대방역세권 등) +
문래동 2개 구역(대선제분1구역, 문래동4가) 등 최소 7건의 대규모 정비사업이
동시 진행 중이나, 이를 종합 안내하는 구청 창구는 확인되지 않음. spec/07의
실제 민원(86240, 공동주택관리법·개인정보보호법 준수 검토 요청, 주택과)은
정비사업 관련 행정절차에 대한 구민의 문제의식을 보여주는 간접 정황으로
해석됨.

[제안 내용]
1. 7개 이상 정비구역을 지도 기반으로 시각화하고, 구역별 현재 단계(정비구역
지정/조합설립/시공사선정/사업시행인가/착공 등)를 표준화된 단계값으로 표시
2. 각 구역 페이지에 담당부서·문의처·최근 진행상황(예: 문래동4가 시공사
선정단계, 신길1구역 현대건설 수주 등)을 정기 갱신
3. 문래동4가(spec/03-2, 8,470억원 총사업비 확인)처럼 예산이 구체적으로
공개된 사업은 IT-1(예산공개플랫폼)과 연동해 함께 조회 가능하도록 설계

[기대효과]
- 정비사업 관련 정보 비대칭 해소
- 대규모 정비사업(7개 구역)에 대한 주민 신뢰도 제고

[IT-4] 문래창작촌-재개발 공존 모니터링 대시보드
=================================================
우선순위: 중
추정예산: 120~200백만원
제안부서(추정): 문화체육과 + 도시계획과

[문제 정의]
spec/05-4, spec/03-1·3-2: 문래동 예술창작촌·철공소거리와 대선제분1구역·
문래동4가 재개발사업이 같은 지역에서 동시 진행 중임이 확인됨. 창작촌·철공소의
존속 방식(이전/재배치/보존)에 대한 구체적 계획은 확인되지 않음. 단, 이는
실제 갈등 사례가 보도된 것이 아니라 조사자가 spec/03과 spec/05의 교차 확인을
통해 도출한 구조적 리스크임을 명확히 함(추정, 확정된 문제가 아님).

[제안 내용]
1. 문래창작촌·철공소거리 입주 현황(작업실 수, 업종 등)을 정기 조사해 재개발
진행 단계(IT-3 대시보드)와 함께 모니터링
2. 재개발구역과 창작촌·철공소거리의 경계가 실제로 겹치는지, 겹친다면 어느
정도 규모인지부터 확인하는 실태조사를 우선 시행(현재 spec 조사로는 물리적
경계 중첩 여부 자체가 확인되지 않았으므로, 본 사업의 1단계는 반드시 실태조사)
3. 실태조사 결과 실제 리스크가 확인될 경우, 대체 공간 마련 등 후속 대책을
검토(현재로서는 리스크 존재 자체가 확정되지 않아 구체적 대책 설계는 시기상조)

[기대효과]
- 리스크가 실재할 경우 조기 대응 가능, 실재하지 않을 경우(창작촌·철공소가
재개발구역과 무관한 위치임이 확인될 경우) 불필요한 우려 해소
