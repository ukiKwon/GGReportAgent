IT/디지털 기획 사업 제안 (필수 포함 항목)
=================================================

=================================================
[IT-1] 도봉구 통합 사업정보 검색 플랫폼 구축
=================================================
우선순위: 최우선
추정예산: 450~600백만원 (1회성 구축비 300~400 + 연 운영비 50~100 × 2년)
제안부서(추정): 정보통신과 + 기획예산과 (구축), 전 부서(콘텐츠 제공)

[문제 정의]
spec/07 조사 결과, 4개년계획 121개 사업 중 약 27%(24개)가 도봉구청 홈페이지 및
구글 검색으로 확인되지 않음. 원인은:
1. 사업이 여러 서브도메인에 분산 (dobong.go.kr, m.dobong.go.kr/majorWork,
   ceo.dobong.go.kr, health.dobong.go.kr, welfare.dobong.go.kr, sd.dobong.go.kr 등
   최소 6개 이상 도메인)
2. 정책문서(4개년계획 PDF)상 사업명과 실제 홈페이지 게시물 제목이 불일치
3. m.dobong.go.kr/majorWork 같은 모바일 전용/JS 렌더링 페이지는 검색엔진 색인 불완전

[제안 내용]
1. 기존 "주요사업 추진현황"(m.dobong.go.kr/majorWork) DB를 데이터 소스로 재활용하되,
   PC/모바일 반응형 웹으로 재구축하고 검색엔진 색인 가능하도록 서버사이드 렌더링(SSR) 적용
2. 4개년계획 121개 사업 전체를 표준 메타데이터(사업명, 정책문서상 코드번호 예: 2-18,
   담당부서, 예산, 추진현황, 관련 페이지 링크)로 정리한 마스터 DB 구축
   - 정책문서 코드(예: "3-12 GTX-C 도봉구 전구간 지하화")를 검색 키워드로 명시적 등록해
     주민이 4개년계획 PDF와 홈페이지를 대조 검색할 때 매칭되도록 함
3. 검색 UX: 사업명/키워드/분야(5대 정책목표)/담당부서/추진상태(계획중·추진중·완료) 필터 제공
4. 각 사업 페이지에 관련 서브도메인 페이지(건강도봉, 도봉복지로 등)를 링크로 통합 노출
5. 열린데이터광장(data.dobong.go.kr) API 연동으로 예산 집행률 등 정량 데이터 자동 표시

[기술 스택 권장]
- 검색엔진: Elasticsearch 또는 경량 Meilisearch (오픈소스, 저비용)
- 프론트엔드: 반응형 웹(React/Vue), SSR 적용 (Next.js 등)로 SEO 대응
- 백엔드: 기존 열린데이터광장 API 활용, 신규 개발 최소화

[기대효과]
- 121개 사업 확인율 27% 미확인 -> 목표 5% 이하로 개선
- 주민의 "우리 동네 사업이 뭐가 있지?" 질문에 단일 창구로 응답 가능
- 향후 4개년계획 개정 시에도 유지보수 용이(마스터DB 갱신만으로 전체 반영)

=================================================
[IT-2] 민원 처리 현황 통합 대시보드 & 알림톡 연동
=================================================
우선순위: 최우선
추정예산: 300~450백만원 (구축비 250~350 + 연 알림톡 발송비 50~100)
제안부서(추정): 감사담당관(옴부즈만 소관) + 정보통신과

[문제 정의]
spec/08 조사 결과:
1. 민원 게시판이 최소 4개 이상 서로 다른 시스템(ceo.dobong.go.kr bbs, WDB_DEV/Receipt,
   WDB_DEV/VOH_RIGHTS, WDB_DEV/OMBUDSMAN, 규제개혁신고센터)에 분산되어 있어
   주민이 자신의 민원을 어느 채널에 넣었는지, 어디서 진행상황을 봐야 하는지 혼란
2. "구청장에게 바란다"는 목록에 "답변완료" 표시가 없어 접수/처리중만 노출, 처리결과 추적 어려움
3. 반면 옴부즈만 채널은 10건 전부 "완료"로 처리속도가 빠름 -> 채널 간 처리속도 격차가
   존재하나 주민은 이 차이를 알 방법이 없어 비효율적 채널에 민원을 반복 제출할 가능성

[제안 내용]
1. 4개 민원 채널(구청장에게 바란다/옴부즈만/규제개혁신고센터/인권침해신고)의 접수번호를
   통합 관리하는 백엔드 대시보드 구축 - 담당공무원이 채널 구분 없이 단일 화면에서 처리
2. 주민용 "내 민원 조회" 페이지: 접수번호+휴대폰 인증으로 채널 무관하게 진행상황 실시간 확인
   (접수 -> 담당부서 배정 -> 검토중 -> 답변완료 4단계 공통 상태값 정의)
3. 카카오 알림톡 연동: 상태 변경 시(답변완료, 담당자 지정 등) 자동 알림 발송
   - 특히 "구청장에게 바란다"처럼 목록에 진행상황이 안 보이는 채널에 우선 적용
4. 처리기간 SLA 설정 및 대시보드 상 지연 건 자동 하이라이트(관리자용)
5. 연 1회 "민원처리 통계 공표"를 옴부즈만 자료실 형태(기존 "운영상황 공표" 관행 활용)로
   자동 생성 - 채널별 평균 처리일수, 유형별 건수 등 투명 공개

[기술 스택 권장]
- 기존 시스템(WDB_DEV, ceo.dobong.go.kr) API/DB 연동 어댑터 개발(각 사이트 구조가 달라
  전면 교체보다 연동 방식이 저비용)
- 알림톡: 카카오 비즈메시지 API (지자체 다수 도입 사례로 검증된 방식)
- 대시보드: 저코드(Low-code) 툴 또는 기존 새올전자민원 인프라 확장 검토

[기대효과]
- 민원 처리 투명성 확보로 옴부즈만 수준(완료율 100%)의 대응속도를 전 채널로 확산
- 2026년 최다 민원인 창동역 관련 민원(구청장에게 바란다 다수 접수, 처리중 다수)의
  처리상황을 주민이 직접 확인 가능해져 반복 민원 감소 기대
- 채널 파편화 해소로 향후 민원 데이터 기반 정책 수립(어느 동/분야에 민원이 집중되는지) 가능

[IT-3] 창동역 일대 공사구간 스마트 보행안전 시스템
=================================================
우선순위: 상
추정예산: 180~250백만원
제안부서(추정): 교통행정과 + 도시관리국(창동역 복합환승센터 TF)

[문제 정의]
spec/08: 2026년 "구청장에게 바란다" 최신 10건 중 과반이 창동역 4번출구 보행로 신설,
창동민자역사 출입구 관련 민원. spec/04: 창동역 복합환승센터(49,500백만원, 2025-2029),
GTX-C 지하화(2,631,000백만원, 2024년부터 본격 투자) 등 장기 대형 공사가 진행 중이나,
4개년계획 세부사업 목록에는 공사 기간 중 임시 보행 안전/우회 안내에 관한 명시적 항목이 없음.
즉 "완공 후" 계획은 있으나 "공사 중" 대응은 계획 공백 상태.

[제안 내용]
1. 창동역 반경 500m 공사구간에 스마트 보행 안내 시스템 구축:
   - QR코드/NFC 기반 실시간 우회로 안내판 (공사 진척에 따라 원격 갱신 가능한 전자게시판)
   - 기존 CCTV 통합관제센터(spec/07 IT-3 근거: CCTV 스마트안전도시 확인됨) 인프라 활용해
     혼잡도 감지 후 안내 문구 자동 전환
2. "구청장에게 바란다"에 접수된 창동역 관련 민원을 전용 태그로 분류, IT-2 대시보드와 연동해
   주간 단위로 공사 관계자(SH공사, 시공사)에 자동 취합 전달하는 프로세스 수립
3. 모바일 웹(반응형) "창동역 공사현황 안내 페이지" 신설 - 현재 공정률, 예상 완료시점,
   우회로 지도를 주 1회 갱신 (기존 majorWork DB의 lidx=550, 567 페이지와 연동)

[기대효과]
- 최다 민원 유형에 대한 선제적 정보 제공으로 반복 민원 감소
- 대형 SOC 공사의 "체감 불편"을 줄여 장기사업에 대한 주민 수용성 제고

=================================================
[IT-4] 소규모 생활사업 홍보 자동화 (SNS/지도 매시업)
=================================================
우선순위: 중
추정예산: 80~120백만원
제안부서(추정): 홍보담당관 + 정보통신과

[문제 정의]
spec/07: 골목상권 간판개선(예산 2,250백만원 배정, spec/03), 언덕길 열선 및 보행환경 개선
(9,162백만원), 반려견 놀이터, 성인문해교육 등 예산이 명확히 배정된 사업들이 검색조차 안 됨.
대형사업(확인율 83~88%)과 소규모 생활사업(확인율 상대적으로 낮음)간 홍보 격차가 뚜렷함.

[제안 내용]
1. IT-1 통합검색 플랫폼과 연동해, 준공/설치 완료된 소규모 사업(간판개선 거리, 열선 설치
   구간, 반려견 놀이터 위치 등)을 지도 기반으로 시각화하는 "우리동네 변화지도" 페이지 구축
2. 완료 시점마다 자동으로 SNS(구 공식 인스타그램/블로그) 카드뉴스 템플릿에 사진+위치+예산을
   채워 넣는 반자동 콘텐츠 생성 워크플로우 도입 (완전 자동화보다 템플릿+검수 방식 권장)
3. 동 주민센터 게시판에도 관할 지역 완료 사업을 자동 노출(열린데이터광장 API 연동)

[기대효과]
- 저비용으로 소규모 사업 체감도 제고, 예산 대비 성과 홍보 강화
- 향후 예산 심의 시 "주민이 체감하는 사업" 근거자료로도 활용 가능
