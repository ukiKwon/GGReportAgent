# IT/디지털 기획 사업 제안

=================================================
[IT-1] 강서구 사업예산 투명공개 플랫폼 구축
=================================================
우선순위: 최우선
추정예산: 380~520백만원(1회성 구축비 260~350 + 연 운영비 60~85 × 2년)
제안부서(추정): 기획예산과 + 스마트도시과(구축), 전 부서(예산 데이터 제공)

[문제 정의]
spec/08 조사 결과, 2026년 강서구 6대 전략분야 45개 사업 중 개별 사업 단위로
구체적 예산액이 확인된 사업은 8건(약 18%)뿐이며, 특히 균형발전(spec/02, 8건)과
교육문화(spec/06, 9건) 2개 분야는 예산이 단 1건도 확인되지 않음. 원인은:
1. 2026년도 예산안(1조4,356억원) 보도자료가 "6대 핵심전략분야 대표사업 나열"
방식이라 개별 사업별 배분액이 대부분 공개되지 않음
2. 서울시 열린재정 포털(openfinance.seoul.go.kr/gangseo)의 "사업 및 예산정보"
페이지가 실제 존재하나, 조사 시점 접속 결과 "등록된 데이터가 없습니다"로 표시되어
0건/0원으로 나타남(spec/08에서 직접 접속 확인) - jongno 조사에서도 동일 현상이
보고되어 서울시 열린재정 포털 자체의 구조적 한계로 추정됨
3. 6대 분야 배정액 합계(11,154억원)와 예산 총액(14,356억원)의 차액 3,202억원의
용처가 특정되지 않음(spec/01)

[제안 내용]
1. 서울시 열린재정 포털의 "사업별 예산정보" API를 강서구 홈페이지에 임베드하고,
부서명을 몰라도 사업명/키워드로 검색 가능한 자체 검색 UI를 구청 페이지에 재구성
(열린재정 포털 원본은 그대로 두되 강서구 접점을 새로 설계 - jongno IT-1과 유사한
접근을 강서구 데이터에 맞춰 적용)
2. 6대 전략분야(균형발전/안전안심/미래경제/복지건강/교육문화/AI행정혁신) 필터 및
사업별 상세페이지(사업명, 담당부서, 예산현액, 집행률, 진행상태) 표준 양식 마련
3. 6대 분야 배정액 합계와 예산 총액의 차액(3,202억원)에 해당하는 일반행정운영비·
인건비 등 미분류 항목도 최소한 "분류명+총액" 수준으로는 공개해 예산 전체상을
구민이 파악할 수 있도록 함
4. 예산안 보도자료에 나열된 사업명(예: "화곡동 수직형 승강편의시설", "강서역사
문화관" 등)을 실제 예산 항목과 1:1 매핑하는 마스터 코드 부여

[기술 스택 권장]
- 백엔드: 서울시 열린재정 포털 API 연동 우선(신규 개발 최소화)
- 프론트엔드: 반응형 웹, 검색엔진 색인 가능하도록 서버사이드 렌더링 적용
- 2026년 하반기 마곡 통합신청사 개청 시점에 맞춰 신청사 내 "열린도서관"
(spec/06(6-2), AI특화서가 도입 예정)의 디지털 인프라와 연동 검토

[기대효과]
- 45개 사업 중 예산 확인율 18%(8/45) -> 목표 70% 이상으로 개선
- "얼마가 어디에 쓰였는지" 구민 문의에 단일 창구로 응답 가능
- 재정자립도 19.4%(spec/01)로 국비·시비 매칭 의존도가 높은 구조에서, 매칭재원의
출처·비중까지 투명하게 공개해 재정 신뢰도 제고

=================================================
[IT-2] 마곡 통합신청사 개청 연계 스마트 민원·서비스 전환 플랫폼
=================================================
우선순위: 최우선
추정예산: 300~450백만원(구축비 240~350 + 연 운영비 30~50 × 2년)
제안부서(추정): 총무과 + 스마트도시과

[문제 정의]
spec/02(2-4,2-5): 2026년 하반기 마곡 통합신청사(연면적 5만9,377㎡, 구청·구의회·
보건소·주민편의시설 통합) 및 등촌2동 주민센터가 함께 이전 개청 예정이나, 개별
사업비는 확인안됨. spec/07(7-1): 이미 AI 민원챗봇이 구 홈페이지에 도입되어 있으나,
신청사 개청이라는 물리적 전환과 기존 AI챗봇 등 디지털 서비스가 어떻게 연동될
계획인지는 spec 조사에서 확인되지 않음. spec/06(6-2): 신청사 내 "열린도서관"에
AI특화서가 도입 예정이라는 점에서, 신청사 자체가 이미 디지털 전환을 표방하고
있음에도 민원 서비스 이전 계획과의 연계는 불분명함.

[제안 내용]
1. 구청·보건소·주민센터가 물리적으로 통합되는 시점에 맞춰, 기존에 부서별로
분산되어 있었을 민원 접수·처리 이력을 신청사 개청과 동시에 단일 시스템으로
통합(spec/07(7-1) AI 민원챗봇을 신청사 통합 민원창구의 1차 응대 채널로 확장)
2. "구청장에게 바란다"(spec/09) 등 기존 민원 채널과 신청사 방문 민원(현장 접수)을
동일한 접수번호 체계로 통합 관리
3. 신청사 이전 과도기(이전 준비~개청 초기) 동안 발생할 수 있는 민원(주소 변경,
업무 공백 등)을 별도로 태깅해 우선 대응
4. 등촌2동 주민센터 이전(spec/02(2-5))에 따른 주민 안내를 AI챗봇 FAQ에 우선
반영해 이전 초기 혼란 최소화

[기대효과]
- 신청사 개청이라는 일회성 전환점을 활용해 디지털 전환 비용을 절감(신규 시스템을
별도로 구축하기보다 이전 작업에 편승)
- AI행정혁신 분야(spec/07, 예산 13억원으로 6대 분야 중 최소)의 기존 자산(AI챗봇)을
재활용해 예산 효율성 확보
- IT-1(예산공개플랫폼)과 연동해 신청사 관련 예산 집행도 투명하게 공개 가능

[IT-3] 구청장에게 바란다 처리현황·공개기준 투명화 대시보드
=================================================
우선순위: 상
추정예산: 180~280백만원
제안부서(추정): 감사담당관 + 스마트도시과

[문제 정의]
spec/09 조사 결과:
1. 2026년 7월 최신 게시글 10건(번호 2179~2169)이 전부 "처리중" 상태였고, 그중
5건이 "비공개"로 표시됨. 무엇이 공개/비공개를 가르는 기준인지는 spec 조사에서
확인되지 않음(민원인이 선택하는 것인지, 내용에 따라 구청이 판단하는 것인지 불명)
2. 페이지네이션(curPage) 파라미터로 2페이지 이후 게시글에 접근을 시도했으나
동일한 1페이지 내용만 반복 반환되는 현상이 확인됨(spec/09) - 조사 도구의 한계일
가능성과, 실제 구민이 사용하는 화면에서도 유사한 문제가 있을 가능성을 모두
배제할 수 없음
3. 게시판 안내문구상 "근무일 기준 7일 이내 처리, 최장 7일 추가 연장 가능"이라는
처리기한 정책은 확인되나, 실제 처리 소요일수 통계는 공개되지 않음

[제안 내용]
1. 공개/비공개 기준을 명문화해 게시판 안내에 게시(개인정보 포함 여부, 특정 시설·
개인 관련 여부 등 기준 명확화)
2. 페이지네이션 접근성 재점검 - 실제 구민이 과거 게시글을 정상적으로 열람할 수
있는지 별도 UI 테스트를 통해 확인(본 조사에서 발견된 접근 한계가 조사 도구
자체의 문제인지, 사이트 구조의 문제인지부터 먼저 규명 필요)
3. "현재 처리중 몇 건", "평균 처리일수" 등 익명화된 통계를 대시보드 형태로
공표해 접수했지만 아직 결과가 나오지 않은 민원의 존재를 구민이 인지할 수 있도록 함
4. IT-2(신청사 개청 플랫폼)와 연동해 신청사 개청 이후 민원처리 체계 개편의
효과를 처리일수 통계로 추적

[기대효과]
- 처리중·비공개 편중 현상에 대한 구민 신뢰도 제고
- 신청사 개청 전후 민원처리 효율성 변화를 객관적으로 측정 가능
- 향후 IT-1(예산공개플랫폼)과 연동해 "이 민원이 어느 예산 사업과 관련있는지"까지
연결 가능(jongno IT-2 제안과 유사한 확장 방향)
