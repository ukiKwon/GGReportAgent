# IT/디지털 기획 사업 제안 (필수 포함 항목)

=================================================
[IT-1] 서초구 사업예산·진행상황 통합공개 플랫폼 구축
=================================================
우선순위: 최우선
추정예산: 350~500백만원(1회성 구축비 250~350 + 연 운영비 50~75 × 2년)
제안부서(추정): 기획예산과 + 스마트도시과(구축), 전 부서(데이터 제공)

[문제 정의]
spec/07 조사 결과, 2026년 서초구 5대 분야 27개 사업 중 예산액이 구체적으로 확인된
사업은 0건이며, 확인됨(11%)·부분확인(11%)을 합쳐도 진행상황조차 확인 가능한 사업은
6건(22%)에 불과함. 원인은:
1. "2026년 달라지는 구정 47건" 보도자료(spec/01)가 사업명 나열 위주로 구성되어
개별 사업 예산이 전혀 포함되지 않음
2. 서울시 열린재정 포털의 "사업 및 예산정보"(openfinance.seoul.go.kr/seocho) 페이지가
실제 존재하나(spec 조사 확인), 종로구 사례와 유사하게 조건 입력 없이는 첫 화면에서
데이터가 조회되지 않는 구조로 추정됨(본 조사에서 세부 페이지 진입까지는 확인 못함)
3. 재정자립도 53.2%(서울 3위)로 재원 여력이 상당함에도 불구하고, 이 재원이 구체적으로
어느 사업에 얼마나 투입되는지 구민이 확인할 방법이 사실상 없는 구조

[제안 내용]
1. 서울시 열린재정 포털의 서초구 "사업별 예산정보" API를 서초구 홈페이지에 임베드하고,
사업명/키워드 기반 기본 검색 UI를 구청 자체 페이지에 재구성
2. "2026년 달라지는 구정 47건"에 나열된 사업명(예: 서초복지돌봄재단, 우면열린문화센터
등)을 실제 예산 항목과 1:1 매핑하는 마스터 코드 부여 - 보도자료 문구와 실제 예산과목명이
다를 가능성(spec/07에서 반복 확인된 패턴)을 해소
3. 5개 분야(주민생활/복지지원/보육과교육/환경/도시인프라) 필터 및 사업별 상세페이지
(사업명, 담당부서, 예산현액, 집행률, 진행상태-신규/착공/준공) 표준 양식 마련
4. 우면열린문화센터처럼 이미 준공된 시설은 "완공" 배지를, AICT벨트처럼 지구지정만
완료된 사업은 "행정절차 진행중" 배지를 부여하는 등 진행단계를 시각적으로 구분

[기술 스택 권장]
- 백엔드: 서울시 열린재정 포털 API 연동 우선(신규 개발 최소화)
- 프론트엔드: 반응형 웹, 검색엔진 색인 가능하도록 서버사이드 렌더링 적용

[기대효과]
- 27개 사업 중 예산 확인율 0% -> 목표 60% 이상으로 개선
- 재정자립도가 높은 자치구라는 강점을 "예산이 보이는 행정"으로 구체화
- 향후 47건 외 나머지 사업(spec/07에서 확인하지 못한 20건 포함) 확산 적용 가능

[IT-2] 대형 SOC 공약(AICT벨트·지하화) 주민 정보공개 대시보드
=================================================
우선순위: 상
추정예산: 200~300백만원
제안부서(추정): 도시계획과 + 교통정책과

[문제 정의]
spec/06(5-5,5-6): 민선9기 핵심공약인 양재·내곡 AICT벨트 조성은 ICT 특정개발진흥지구
지정(2026.1)까지 완료되어 확인됨으로 분류된 반면, 경부간선도로·반포대로 지하화는
취임사 수준의 "선언"에 머물러 부분확인으로 분류됨(spec/07). 두 공약 모두 대규모
SOC 성격으로 인접 주민(양재·내곡, 경부고속도로·반포대로 인접지역)의 관심이 클
것으로 예상되나, 진행상황을 종합 안내하는 창구는 확인되지 않음.

[제안 내용]
1. AICT벨트(양재1·2동 ICT 특정개발진흥지구, 111만㎡)의 지구 지정 이후 단계
(인허가, 착공, 입주기업 유치 등)를 표준화된 단계값으로 표시하는 진행현황 페이지 구축
2. 경부간선도로·반포대로 지하화는 아직 설계 단계 이전이므로, "공약 확정 전" 상태를
투명하게 표기하고 향후 타당성조사·설계 착수 시점부터 자동 갱신되는 틀을 사전 설계
3. IT-1(예산공개플랫폼)과 연동해 향후 사업비가 반영되는 즉시 예산 항목과 자동 연결
4. 인접 주민(양재·내곡동, 경부고속도로변 지역) 대상 설명회 일정·자료를 이 페이지에
통합 게시

[기대효과]
- 서울시 차원 도시계획 절차(지구지정 등)와 서초구 자체 홍보 사이의 정보 격차 해소
- 대형 SOC 공약 특유의 "선언은 화려하나 진행상황은 안 보이는" 문제 사전 예방
- FN-1(인접지역 지원금)과 연계해 지원 대상 구역을 객관적 진행단계 기준으로 산정

[IT-3] 민원 처리 부서 정보 복원형 대시보드 (국민신문고 연계 보완)
=================================================
우선순위: 상
추정예산: 150~250백만원
제안부서(추정): 감사담당관 + 디지털행정과

[문제 정의]
spec/08: 2025.6.25 국민신문고 연계 전환 이후, 서초구 홈페이지에서 열람 가능한
게시글 목록에 담당부서가 개별 표기되지 않고 "감사담당관"으로만 총괄 표기됨. 이는
종로구(부서별 표기, 10개 이상 부서 분산도 분석 가능)와 비교했을 때 오히려 구민이
확인할 수 있는 정보가 줄어든 구조임. 표본 10건 중 메이플자이아파트 설계변경 미고지
민원처럼 대형 재건축단지 관련 민원이 확인되어, 서초구 특유의 재건축단지 밀집
특성을 반영한 유형 분석이 필요하나 현재 구조로는 어려움.

[제안 내용]
1. 국민신문고 연계는 유지하되, 서초구 자체 대시보드에서 접수번호 기반으로 처리
부서(익명화 수준: 부서명까지만, 담당자명은 비공개)를 별도로 매핑해 재공개하는
보완 시스템 구축(국민신문고 원본 데이터는 그대로 두고 구청 자체 표시 레이어만 추가)
2. 유사 민원 자동 태깅(재건축단지 관련, 시설보수 관련 등 키워드 기반) - 표본에서
확인된 재건축단지 민원, 체육시설 보수 민원 등 유형을 자동 그룹핑
3. 연 1~2회 "민원처리 통계 공표"(부서별 접수건수, 평균 처리일수 등) 자동 생성
4. 국민신문고 연계 이전(2025.6.25 이전) "(구)구청장에게 바란다" 게시판의 과거
데이터도 통계 목적으로는 통합 집계에 포함하는 방안 검토(개인정보는 마스킹 유지)

[기대효과]
- 국민신문고 통합의 장점(단일 창구)은 유지하면서 종로구 수준의 부서별 투명성 회복
- 재건축단지 밀집이라는 서초구 특성에 맞는 민원 유형 분석 기반 마련
- IT-1과 연동해 "이 민원이 어느 예산 사업과 관련있는지"까지 연결 가능
