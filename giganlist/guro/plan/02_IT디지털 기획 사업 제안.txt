# IT/디지털 기획 사업 제안 (필수 포함 항목)

=================================================
[IT-1] 구로구 사업예산 투명공개 플랫폼 구축
=================================================
우선순위: 최우선
추정예산: 400~550백만원(1회성 구축비 280~380 + 연 운영비 60~85 × 2년)
제안부서(추정): 기획예산과 + 디지털정보과(구축), 전 부서(예산 데이터 제공)

[문제 정의]
spec/07 조사 결과, 2026년 구로구 4대 실행전략 21개 사업 중 개별 예산액이 구체적으로
확인된 사업은 0건임(존재 확인율 62%와 대비되는 예산 확인율 0%). 원인은:
1. 2026년도 예산안(1조 539억원) 보도가 "4대 실행전략 + 대표사업 나열" 방식이라
전략별·사업별 배분액이 공개되지 않음(spec/01)
2. 서울재정포털 구로 페이지(openfinance.seoul.go.kr/guro)의 "사업별 예산정보" 조회는
실제 존재하나, 조회 시 "등록된 데이터가 없습니다"로 표시되어 일반 구민이 이용하기
어려움(spec/01, 직접 접속 확인)
3. 부서별 세부 예산을 담았을 것으로 추정되는 '2026 구정현황' 전자책(약 240p)은 뷰어
형식이라 검색·추출이 어려움(spec/05 4-5, spec/07 한계)

[제안 내용]
1. 서울재정포털의 "사업별 예산정보" API를 구로구 홈페이지에 임베드하고, 부서명을
몰라도 사업명/키워드로 바로 검색되도록 기본 검색 UI를 구청 자체 페이지에 재구성
(서울재정포털 원본은 그대로 두되 구로구 접점을 새로 설계)
2. 4대 실행전략(사회서비스 확대/주민소득 증대/주택공급/참여자치) 필터 및 사업별
상세페이지(사업명, 담당부서, 예산현액, 집행률, 진행상태) 표준 양식 마련
3. '2026 구정현황' 전자책에 담긴 부서별 예산 데이터를 구조화된 텍스트/표로 재입력해
검색 가능한 형태로 공개(전자책과 병행)
4. 예산안 보도자료 사업명(예: "발달장애인 배상책임보험", "신도림역 구로1유수지 공영
주차장")을 실제 예산 항목과 1:1 매핑하는 마스터 코드 부여

[기술 스택 권장]
- 백엔드: 서울재정포털 API 연동 우선(신규 개발 최소화), 예산서 원문(guro.go.kr
예산서 게시판 bbsNo=657) 파싱 보조
- 프론트엔드: 반응형 웹, 검색엔진 색인 가능하도록 서버사이드 렌더링 적용

[기대효과]
- 21개 사업 중 예산 확인율 0% -> 목표 70% 이상으로 개선
- "얼마가 어디에 쓰였는지" 구민 문의에 단일 창구로 응답 가능
- 낮은 재정자립도(19.9%) 여건에서 재원 배분 투명성 제고로 정책 신뢰도 향상

[IT-2] 정비사업 진행상황 통합안내 지도
=================================================
우선순위: 상
추정예산: 250~350백만원(구축비 200~280 + 연 운영비 40~60)
제안부서(추정): 도시재생과 + 주택과

[문제 정의]
spec/04: 구로구는 가리봉1구역(2,259가구, 임대 609 포함), 고척동253(1,469가구),
개봉·구로동 신통기획 후보지, 길훈아파트 재건축 등 대규모 정비사업이 동시다발로
진행 중이며, 이는 G밸리 배후 청년·1~2인 가구 주거와 직결됨(spec/04, 3-2). 그러나
구역별 진행단계·이주·분담금 정보를 종합 안내하는 창구는 확인되지 않음. 정보 비대칭이
민원으로 이어질 개연성이 크나, spec/08에서 실제 민원 게시글을 확인하지 못했으므로
이는 "개연성 수준의 추정"임(사실로 단정하지 않음).

[제안 내용]
1. 구로구 정비구역 전체를 지도 기반으로 시각화하고, 구역별 현재 단계(정비계획 수립/
정비구역 지정/조합설립/사업시행인가/관리처분/착공/입주 등)를 표준화된 단계값으로
표시하는 "구로 정비사업 진행현황 지도" 구축
2. 서울시 정비사업 공식DB(news.seoul.go.kr/citybuild, 정보몽땅 cleanup.seoul.go.kr)에
이미 등재된 데이터를 우선 연동해 신규 데이터 구축 최소화
3. 각 구역 페이지에 이주 지원·분담금 상담 안내, 관련 민원 접수 링크를 표준 배치
4. G밸리 직주근접 배후주거(가리봉1구역 등)는 별도 상세페이지로 청년·1~2인 가구 대상
공급 물량·입주 시기를 강조 안내
5. IT-4(민원 통합관리)와 연동해 구역별 민원 발생 현황을 담당부서가 모니터링

[기대효과]
- 정비사업 정보 비대칭 해소로 감사요청성·반복 민원 예방
- 대규모 정비사업(합산 수천 세대 영향권)에 대한 주민 수용성 제고
- GTX-B 환기구(spec/01) 등 개발 현안도 동일 지도에 통합 안내 가능

[IT-3] 구로형 기본사회 데이터 대시보드(100m 격자 연계)
=================================================
우선순위: 상
추정예산: 200~320백만원
제안부서(추정): 복지정책과 + 기획예산과

[문제 정의]
spec/05(4-1): 민선9기는 구민 소득 구조를 100m 격자 단위로 분석해 사회적 약자 지원
정밀도를 높이는 방식을 도입 중이나 구축 예산·시기가 미확정임(부분확인). spec/02:
구로형 기본사회(사회서비스재단·기본사회위원회) 추진체가 설립/구성 준비 단계로 세부
미확정. 즉 데이터 기반 복지 정책의 "설계 단계"이므로, 지금이 데이터 정합성과 투명성을
처음부터 내장할 수 있는 시점임.

[제안 내용]
1. 100m 격자 소득 데이터를 개인정보 비식별화 기준으로 집계해, 복지 사각지대(저소득·
1인가구·발달장애 등) 밀집 격자를 시각화하는 내부 정책 대시보드 구축
2. 구로사회서비스재단(spec/02, 1-3) 설립 시 통합돌봄 서비스 배치의 근거 데이터로 활용
3. 구민 공개용은 개인정보 위험이 없는 수준(동 단위 집계 지표)으로만 공개하고, 정밀
격자 데이터는 내부 정책용으로 분리 - 데이터 윤리·비식별화 원칙을 설계 단계부터 명시
4. IT-1(예산공개플랫폼)과 연동해 "어느 지역에 어떤 복지예산이 배분되는지"를 데이터로
검증 가능하게 함

[기대효과]
- 구로형 기본사회 정책의 정밀 타겟팅으로 취약계층 지원 효율 제고
- 데이터 기반 행정의 투명성·재현성 확보(설계 단계 내장)

=================================================
[IT-4] 민원 통합관리 & 열린구청장실 접근성 개선
=================================================
우선순위: 중
추정예산: 200~300백만원
제안부서(추정): 감사담당관 + 디지털정보과

[문제 정의]
spec/08: 구로구 실사용 민원 채널은 "구청장에게 바란다"(evpwcList.do)와 옴부즈맨
소관 "고충민원신청" 등으로 나뉘며, 게시판이 동적 렌더링·비공개 운영이라 2026년 실제
게시글 목록을 외부에서 확인하기 어려움(본 조사도 목록 확인 실패). "처리완료 전 비공개"
운영은 종로구와 유사하나, 처리중 건수·평균 처리기간을 구민이 알 수 없는 불투명성이
있음. spec/05(5-4): 열린구청장실(민원데이트/공약사업현황)이 별도 운영됨.

[제안 내용]
1. "구청장에게 바란다"·고충민원·민원데이트 접수 건을 동일 접수번호 체계로 통합 관리하는
백엔드 대시보드 구축 - 담당 공무원이 채널 구분 없이 단일 화면에서 처리
2. 구민용 "내 민원 조회": 접수번호+휴대폰 인증 기반으로 진행상황 확인, 처리중 건도
익명 통계("현재 처리중 N건")로 공개해 불투명성 해소
3. 유사 민원 자동 태깅(재개발/주차/환기구 등 키워드)으로 중복 대응 방지 및 정책화
우선순위 판단 지원
4. 연 1~2회 민원처리 통계 공표(부서별 접수건수, 평균 처리일수)
5. IT-1·IT-2와 연동해 민원이 어느 예산 사업·정비구역과 관련있는지 연결

[기대효과]
- 민원 접근성·투명성 개선, 반복·중복 민원 감소
- spec/08에서 확인 못한 민원 데이터 공백을 구조적으로 해소(향후 조사 가능성 제고)
