# IT/디지털 기획 사업 제안 (필수 포함 항목)

=================================================
[IT-1] 중랑구 홈페이지 접속안정성·표준 프로토콜 개선
=================================================
우선순위: 최우선
추정예산: 150~250백만원(진단·개선 컨설팅 80~120 + 서버/네트워크 구성 개선
70~130)
제안부서(추정): 디지털행정과/정보통신팀 + 외부 웹표준·보안 컨설팅

[문제 정의]
spec/00·spec/08 조사 결과, 중랑구청 홈페이지(jungnang.go.kr)는 본 조사에서
서로 다른 프로토콜(http/https), 경로(루트/sitemap/서브페이지), 우회수단(프록시,
아카이브)으로 9회 이상 접속을 시도했으나 전부 실패했음(주로 "Parse Error:
Unexpected space after start line"라는 HTTP 응답 형식 오류). 이는:
1. 표준 HTTP 클라이언트(조사에 사용한 WebFetch 도구 포함)가 정상적으로 파싱하지
못하는 형식의 응답을 반환하고 있다는 뜻이며, 검색엔진 크롤러 역시 유사한 어려움을
겪어 검색 노출도가 떨어질 가능성이 있음(추정 - 확정적 인과관계는 아님)
2. 실제로 본 조사에서 jungnang.go.kr의 게시글이 검색엔진(Google 뉴스 RSS 포함)에
전혀 노출되지 않는 현상이 관찰되었는데, 이것이 접속장애와 관련이 있는지는
확인안됨이나 정황상 개연성이 있음

[제안 내용]
1. 홈페이지 서버의 HTTP 응답 헤더/상태줄 형식이 표준(RFC 7230 등)을 준수하는지
외부 진단 도구로 점검(자체 브라우저에서는 정상으로 보이더라도, 표준 파서 기준
비표준 응답을 반환하는 경우가 실제로 존재함)
2. 웹방화벽(WAF)·보안장비가 있다면, 정상적인 자동화 도구(검색엔진 크롤러,
접근성 진단 도구 등)의 요청까지 과도하게 차단·변형하고 있지 않은지 점검
3. 사이트맵(sitemap.xml) 정상 노출 여부 점검 - 본 조사에서 sitemap 경로가
404로 확인되어, 검색엔진 색인 효율에 영향이 있을 수 있음
4. 개선 후 정기적으로(분기 1회) 외부 표준 진단 도구로 접속 상태를 자체 점검하는
프로세스 마련

[기대효과]
- 구민의 웹 접근성 및 검색엔진을 통한 정보 접근성 개선
- 향후 IT-2~IT-4의 모든 디지털 서비스가 정상 작동하기 위한 전제조건 확보
- 외부 조사기관·타 기관의 정보공개 요청 대응 시에도 접근성 문제로 인한 불필요한
민원 예방

[IT-2] 중랑구 사업예산 투명공개 플랫폼 구축
=================================================
우선순위: 최우선
추정예산: 400~550백만원(구축비 300~400 + 연 운영비 50~75 × 2년)
제안부서(추정): 기획예산과 + 디지털행정과(구축), 전 부서(예산 데이터 제공)

[문제 정의]
spec/07 조사 결과, 2026년 중랑구 5대 분야 34개 사업/항목 중 예산액이 구체적으로
확인된 사업은 2건(면목2동 출렁다리 150억원, 복합청사 400억원)뿐이며, 이마저도
단일 매체·의원 발언 인용 수준으로 신뢰도가 제한적임(사실상 구청 공식 예산문서
기준으로는 34건 전체가 확인안됨에 가까움). 원인으로 추정되는 사실:
1. 2026년도 예산 총액 자체도 시차를 둔 두 개의 수치(1조1,648억원 vs 1조1,252억원,
spec/01)로 보도되어, 총액 수준의 정보조차 일관되게 공개되지 않음
2. IT-1에서 지적한 홈페이지 접속장애로 인해, 설령 예산 관련 정보가 게시되어 있다
하더라도 외부에서 확인하기 어려운 구조임

[제안 내용]
1. 서울시 열린재정 포털(openfinance.seoul.go.kr) 등 서울시 공통 시스템의 중랑구
데이터를 구 홈페이지에 임베드하고, 사업명/키워드 기반 검색 UI를 별도 구성
2. 34개 사업/항목 중 이번 조사로 확인된 사업명(예: 면목선 경전철, 중랑물재생센터
현대화, 상봉9구역 재개발 등)을 실제 예산과목명과 1:1 매핑하는 마스터 코드 부여
3. 보도자료에 등장한 두 가지 예산 총액 수치(1조1,648억원/1조1,252억원)의 차이가
어떤 절차(편성안->심의->확정)에 따른 것인지 명확히 설명하는 예산단계 안내 페이지
신설
4. 5대 분야(도시재생주택교통/복지건강/교육청년/문화관광인프라/경제일자리재정)
필터 및 사업별 상세페이지(담당부서, 예산현액, 집행률, 진행상태) 표준 양식 마련

[기대효과]
- 34개 사업 중 예산 확인율 약 6%(2/34, 그마저도 신뢰도 낮음) -> 목표 70% 이상
개선
- IT-1 개선과 결합해 실질적으로 외부에서 접근 가능한 투명성 확보

[IT-3] 정비사업(재건축·재개발) 진행상황 통합 안내 시스템
=================================================
우선순위: 상
추정예산: 200~300백만원
제안부서(추정): 도시재생과 + 주택정책과

[문제 정의]
spec/02: 중랑구는 상봉9구역(999세대, 포스코이앤씨 시공, 2029.5 준공목표),
상봉7구역(841세대), 중화6구역, 면목10구역 등 최소 4개의 정비구역이 동시에 진행
중이나, 확인등급이 확인됨 1건(상봉9구역)·부분확인 3건(나머지)으로 갈려 정비구역
간 정보 공개 수준에 편차가 있음이 확인됨. spec/08에서 밝힌 대로 민원게시판
자체에 접근하지 못해 종로구의 "창신23정비사업 감사요청" 같은 구체적 민원 사례는
확보하지 못했으나, 이는 오히려 "정보 접근성이 더 나쁠 가능성을 배제할 수 없다"는
뜻으로 해석함(추정).

[제안 내용]
1. 4개 정비구역(상봉9·상봉7·중화6·면목10)을 포함해 향후 신규 지정되는 구역까지
지도 기반으로 시각화하고, 구역별 현재 단계(기본계획수립/구역지정/조합설립/
사업시행인가/관리처분/착공/준공)를 표준화된 단계값으로 표시
2. 상봉9구역처럼 정보가 비교적 풍부한 구역을 표준 템플릿으로 삼아, 상봉7·중화6·
면목10구역도 동일 수준의 정보(세대수, 시공사, 준공목표, 견본주택 일정 등)를
공개하도록 표준화
3. IT-1(홈페이지 접속안정성 개선)이 선행되어야 이 시스템도 정상적으로 구민에게
노출될 수 있음을 전제조건으로 명시
4. spec/02에서 확인된 모아주택(금천·강북·중랑 공동, 3,867가구) 등 광역 사업도
연동 표시

[기대효과]
- 4개 정비구역 간 정보공개 편차 해소
- 대규모 주택정비(4개 구역 합산 최소 2,000세대 이상 영향권 추정)에 대한 주민
신뢰도 제고

[IT-4] 민선9기(중랑동행 비전위) 공약이행 추적 플랫폼
=================================================
우선순위: 상
추정예산: 150~250백만원
제안부서(추정): 기획예산과 + 홍보과

[문제 정의]
spec/01: 류경기 구청장(3선, 2026.6 당선·2026.7.1 취임)이 신임이 아닌 연임임에도
불구하고 "중랑동행 비전위원회"를 별도로 출범시켜 주민과 함께 공약을 구체화하는
절차를 조사 시점에도 진행 중임이 확인됨. 이는 종로구의 "신임 구청장 취임 초
공약 미확정" 상황과는 배경이 다르지만, 공약이 아직 확정 문서화되지 않은 시점이라는
점에서는 유사하게 투명성을 초기 설계에 반영할 수 있는 기회임.

[제안 내용]
1. 중랑동행 비전위원회의 공약 구체화 결과가 확정되는 즉시 이행 단계(공약확정 ->
실천계획수립 -> 예산반영 -> 착수 -> 완료)를 구민이 추적할 수 있는 페이지를
사전 설계
2. IT-2(예산공개플랫폼)와 동일한 코드체계로 예산이 반영되는 즉시 자동 연동되도록
설계
3. 비전위원회가 이미 3선 연임 구청장의 정책 연속성 위에서 출범한 만큼, 기존
민선8기 사업(spec/02~06에서 확인된 34개 항목 다수가 사실상 8기 예산으로 집행
중인 사업)과 신규 공약을 구분해 표시하는 "연속사업/신규공약" 태그 기능 포함

[기대효과]
- 공약 구체화 단계부터 투명성을 내장해 "예산 따로, 공약 따로" 문제 예방
- 3선 연임이라는 정책 연속성의 장점(신임 구청장보다 빠른 실행력 기대)을 투명성
있게 구민에게 소통하는 채널 확보
