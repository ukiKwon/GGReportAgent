# IT/디지털 기획 사업 제안 (필수 포함 항목)

=================================================
[IT-1] 용산구 예산-사업 매핑 및 PDF 접근성 개선 플랫폼
=================================================
우선순위: 최우선
추정예산: 400~550백만원(1회성 구축비 280~380 + 연 운영비 60~85 × 2년)
제안부서(추정): 기획조정실(기획예산담당관) + 스마트정보과(구축)

[문제 정의]
spec/09[G] 조사 결과, 용산구 홈페이지에 실제로 "2026년 주요업무계획" PDF(60.8MB)가
2018년부터 매년 누락 없이 게시되어 있고 다운로드까지 정상적으로 되지만, 한글 폰트가
유니코드로 매핑되어 있지 않아 텍스트 추출이 원천적으로 불가능함이 확인됨(pdftotext
시도 결과 한글 0단어 추출). 이는 "자료 부재"가 아니라 "자료는 있는데 읽을 수 없는"
더 교묘한 형태의 불투명성으로, PDF 목차상 6개 분야 구조(PART 4, p.19/31/43/53/61/71
시작)까지 확인되었음에도 각 분야의 한글 제목조차 알 수 없는 상황임.
아울러 spec/00·09[H]에서 2026년도 예산 수치가 두 가지(사업예산안 6,770억원 vs
PDF 그래프상 1,030,312, 단위미상)로 불일치함이 확인되어 예산 총액 자체의 신뢰성에도
의문이 있음.

[제안 내용]
1. "주요업무계획" PDF를 신규 발간분(2027년분)부터 텍스트 레이어가 정상 매핑된
형태(ToUnicode CMap 포함 폰트 임베딩) 또는 HWP/텍스트 병행 게시로 전환 - 문서
작성 시스템(한글과컴퓨터 등)의 PDF 변환 설정 점검을 정보화 담당부서에 요청
2. 리플릿(용산구보 웹진, spec/02~06의 55개 사업 출처)과 회의록(yscl.go.kr) 원문을
사업명 기준으로 1:1 매핑하는 마스터 코드 부여 - "예정일"(리플릿)과 "실제 시행일"
(언론보도)이 다른 사례(spec/03·04에서 3건 확인)를 시스템적으로 최신화
3. 6,770억원(사업예산안)과 PDF 내 1,030,312(단위미상) 수치 간 불일치의 실제 원인
(세입세출 총규모 vs 사업예산안 등 포함범위 차이 여부)을 기획예산담당관에 공식
질의하여 명확화하고, 플랫폼에 "왜 두 숫자가 다른지"를 명시적으로 설명하는 FAQ 페이지
게시
4. 서울시 열린재정 포털과의 연동을 검토해 부서별·사업별 예산현액·집행률 조회 기능
제공

[기술 스택 권장]
- 신규 문서 생성 프로세스 개선(한글 문서작성 프로그램의 PDF 내보내기 옵션 표준화)이
최우선 - 과거 발간분(2018~2026) 전체를 재작업하는 것은 예산상 후순위로 배치
- 프론트엔드: 반응형 웹, 사업명 검색 기능 우선 구현

[기대효과]
- "문서는 있는데 못 읽는" 역설적 상황 해소 - 향후 매년 발간되는 주요업무계획의
실질적 활용도 제고
- 6,770억원 vs 1,030,312 수치 불일치에 대한 구민 문의에 명확히 답변 가능한 근거자료
확보

[IT-2] 정비사업 통합 모니터링 대시보드
=================================================
우선순위: 최우선
추정예산: 350~480백만원(구축비 260~350 + 연 운영비 45~65)
제안부서(추정): 재정비사업과 + 도시계획과 + 스마트정보과

[문제 정의]
spec/09[B][C]에서 확인된 대로, 용산구는 한남뉴타운(2~5구역)·이촌동 개별단지
재건축·리모델링('이촌 르엘' 등)·원효로3가1구역 신속통합기획 등 다수의 정비사업이
동시에 진행 중이나, 이 구역들의 진행상황을 종합 안내하는 창구는 확인되지 않음.
spec/08 민원게시판에서 실제로 (1) 이촌르엘 리모델링 시공 관련 임의시공·비리의혹
민원이 2026년 4월 한 달간 반복 게시되었고, (2) 원효로3가1구역에서는 2026.5.2
하루에 9건의 유사 문구 청원이 집중 게시되어 조직적 여론전이 의심되는 정황까지
포착됨. 이는 정비사업 진행 과정의 정보 비대칭이 실제 민원·분쟁 형태로 표출된
구체적 사례임.

[제안 내용]
1. 한남뉴타운 2~5구역(spec/06에서 이미 확인된 구역별 시공사·세대수·인가단계),
이촌동 개별단지(반도아파트·한강맨션·성경아파트·이촌현대아파트), 원효로3가1구역을
포함한 "용산 정비사업 진행현황 지도"를 구축, 표준화된 단계값(기본계획수립/구역지정/
조합설립/사업시행인가/관리처분/착공/입주)으로 시각화
2. 각 구역 페이지에 "민원·분쟁 신고 안내" 및 "공정 여론수렴 가이드"(원효로3가1구역
사례처럼 특정 시점에 유사 문구 청원이 집중될 경우 자동 감지·안내) 링크 배치
3. 이촌르엘류 리모델링 시공 관련 민원은 IT-3(민원 이관추적 시스템)과 연동해
"시공 하자·조합 운영 분쟁" 유형으로 별도 태깅, 주택과·재정비사업과가 공동 모니터링
4. 원효로3가1구역처럼 찬반 여론이 첨예한 구역은 "게시글 집중도"(동일자 다건 게시
등) 통계를 담당부서에 자동 알림해 여론 왜곡 여부를 조기에 인지하도록 지원

[기대효과]
- 정비사업 정보 비대칭 해소로 이촌르엘·원효로3가1구역 유형의 갈등성 민원 조기 포착
- 대규모 정비사업(한남뉴타운 4개구역 합계 약 1만2천여가구)에 대한 주민 신뢰도 제고

=================================================
[IT-3] 민원 이관추적 투명화 시스템
=================================================
우선순위: 상
추정예산: 250~350백만원(구축비 200~270 + 연 운영비 25~40)
제안부서(추정): 민원여권과 + 감사담당관

[문제 정의]
spec/08 조사 결과, "구청장에게 바란다" 게시판 표본(약 90건) 중 상당수가 "게시물이
타 신고접수(응답소) 민원창구로 이동됨"으로만 표시되고 실제 처리 내용이나 결과는
게시판에서 더 이상 추적할 수 없는 구조임이 확인됨. 이는 종로구(확인된 15건 전부
"답변완료"로 공개 - jongno 사례) 대비 용산구가 처리현황 투명성 면에서 뒤처지는
지점으로 판단됨.

[제안 내용]
1. "구청장에게 바란다"에서 응답소 등 외부 채널로 이관된 건에 대해, 이관 후 최종
처리결과(처리완료 여부, 처리부서)까지 원 게시글에 사후 업데이트되는 연동 체계 구축
2. 구민용 "내 민원 조회" 페이지: 접수번호 기반으로 이관 전후 전체 이력을 한 화면에서
확인 가능(접수 → 이관 → (응답소 처리) → 최종결과)
3. 정책 제안형 민원(spec/08의 도서관 건립요청, 영어도서관 신설 등 교육 인프라 요구)은
IT-1의 예산-사업 매핑 코드와 연계해 "이 제안이 실제 사업으로 채택되었는지"까지
장기 추적 가능하도록 설계
4. 연 1회 "민원처리 통계 공표"(이관 비율, 답변완료까지 평균 소요일 등) 자동 생성

[기대효과]
- 이관 후 "블랙박스화"되는 민원의 최종 결과를 구민이 확인할 수 있게 되어 행정
신뢰도 제고
- 원효로3가1구역·이촌르엘 유형처럼 반복되는 민원 패턴을 통계적으로 조기 포착

[IT-4] 직속기구(개발신속추진단·안전재난관리단) 운영성과 공개 플랫폼
=================================================
우선순위: 상
추정예산: 150~250백만원
제안부서(추정): 기획조정실(미래전략담당관) + 홍보담당관

[문제 정의]
spec/09[A]에서 확인된 대로, 김경대 구청장은 2026.7.1 취임 당일 "용산개발
신속추진단"과 "안전재난관리단" 2개 직속기구 신설을 1호 결재로 승인했으나, 두
기구 모두 조사 시점 기준 "임시조직"이며 조례 개정·조직개편 절차가 아직 진행되지
않은 상태임. 종로구 사례(민선9기 신규공약이 취임 초기 "선언" 단계)와 유사하게,
지금이 바로 이 조직들의 운영성과 관리체계에 투명성을 처음부터 내장할 수 있는
시점임.

[제안 내용]
1. 두 직속기구가 정식 직제화되는 즉시(조례 개정 시점에 맞춰), IT-1의 예산-사업
매핑 코드 체계와 동일한 방식으로 조직별 추진과제·예산·성과지표를 공개하는 페이지
사전 설계
2. "용산개발 신속추진단"의 추진 대상(한남뉴타운, 이촌동 정비사업, 원효로3가1구역
등 IT-2에서 이미 다루는 구역들)과 IT-2 대시보드를 연동
3. "안전재난관리단"은 spec/09[D]의 미군기지 유류오염 정밀조사, spec/06의 CCTV·
제설 등 기존 안전건설교통국 사업과의 역할 분담을 명확히 공개
4. 임시조직 단계의 활동(회의 개최 횟수, 현장 방문 등)부터 우선 공개해 "선언"에
그치지 않고 실제 가동 중임을 구민에게 보여주는 효과 기대

[기대효과]
- 신규 직속기구 출범 초기부터 투명성을 내장해 향후 "설치는 했는데 뭘 하는지
모르겠다"는 비판 사전 예방
- 취임 초기 관심도가 높은 시점을 활용해 구민 참여도 제고
