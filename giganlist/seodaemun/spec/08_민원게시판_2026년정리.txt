# 서대문구청 홈페이지 민원 관련 게시판 - 2026년 게시글 조사 결과

조사일 기준: 2026년 7월 20일
조사 대상: 서대문구청 홈페이지(sdm.go.kr) "열린청장실" 하위 민원 게시판

[결론 먼저] 서대문구는 종로구·타 자치구 사례와 달리, 본 조사에서 "구청장에게
바란다" 게시판의 2026년 실제 게시글 목록을 확인하지 못했다(접근불가). 이유는
아래에 상세히 기술한 대로, WebFetch가 해당 페이지에서 매 조회마다 서로 다른
내용을 반환하는 hallucination 현상을 직접 확인했기 때문이다.

[게시판 기본 정보 - 확인됨]
- 명칭: "구청장에게 바란다" (열린청장실 하위)
- URL: https://www.sdm.go.kr/chief/M2_Story/m2_01.do
- 운영방식: "이곳은 국민 개개인이 작성하여 올린 글에 대해 구청에서는 답변 의무가
없으며, 게시물이 본 정책에 위배될 경우 삭제될 수 있습니다"라는 안내문구, 실명제
필수(비실명 시 회신 불가), 접수 후 7일 이내 답변 처리 원칙 - 이 항목들은 3회
조회 중 2회에서 동일하게 나와 "확인됨"으로 처리
- 민원상담 연락처: 02-330-1566 (2회 조회에서 동일하게 나옴 - 확인됨)
- 구청 주소: 서울 서대문구 연희로 248 (연희동) - 위키백과와도 일치, 확인됨

[2026년 실제 게시글 목록 - 접근불가 판정 및 근거]
동일 URL(https://www.sdm.go.kr/chief/M2_Story/m2_01.do)에 대해 서로 다른
프롬프트로 3회 WebFetch를 수행한 결과, 게시글 총수·페이지수(21,913건/2,192페이지)
는 3회 모두 동일하게 나왔으나, 최상단 게시글(번호 21913으로 표기됨)의 "제목"이
매 조회마다 전혀 다른 내용으로 반환되었다:
1차 조회: "민원과 건축허가 관련 질의"
2차 조회(원문 그대로 인용 요청): "도시철도 공사현장 주변 소음 민원"
3차 조회(제목만 답하도록 재요청): "구청인사과 관리직 신규 공개채용 모집 공고.."

동일 게시글 번호에 대해 세 번 모두 다른, 그러나 각각 그럴듯한 민원 제목이
나왔다는 것은 실제 페이지 콘텐츠를 읽은 것이 아니라 WebFetch의 요약 모델이
"민원게시판에 있을 법한 제목"을 매번 새로 지어낸 것으로 판단할 수밖에 없다.
이는 해당 게시판이 JavaScript 기반으로 렌더링되어 WebFetch가 실제 DOM을 읽지
못하고 빈 틀 또는 메타정보만 확보한 상태에서 소형 모델이 그럴듯한 내용을
채워넣은(hallucination) 것으로 강하게 의심됨.

따라서 본 조사는 사용자 지시사항의 "확인되지 않은 사실을 지어내지 마라" 원칙에
따라, 아래 데이터를 전부 채택하지 않고 폐기하였다:
- 개별 게시글 제목·날짜·담당부서·처리상태 목록 전체 (2026-07-20 기준 상위
10건이라며 제시되었던 표 포함)
- "전체 21,913건" 숫자도 3회 모두 동일하게 나오긴 했으나, 제목이 매번 바뀌는
모델의 특성상 이 숫자 역시 실제 페이지값인지 확신할 수 없어 "확인안됨(재현되나
근거 신뢰도 낮음)"으로 별도 표기함

[민원 유형 패턴 분석] - 수행 불가
위와 같은 이유로 2026년 실제 게시글에 기반한 민원 유형 패턴 분석(생활환경/
보행교통/시설공사 등 분류)은 이번 조사에서 수행하지 못했다. 종로구 조사(spec/08)
처럼 15건의 실제 게시글을 유형별로 분류하는 작업은 신뢰할 수 있는 원문 데이터
없이는 fabrication 위험이 있어 시도하지 않았다.

[비고 - 타 자치구 조사와의 방법론적 차이]
종로구·도봉구 등 기존 조사에서는 게시판이 검색엔진에 노출되거나 정적 HTML로
목록이 렌더링되어 WebFetch/WebSearch로 실제 게시글 목록을 직접 확인할 수
있었던 것으로 보이나, 서대문구 "구청장에게 바란다" 게시판은 (1) WebSearch
예산 소진으로 애초에 검색 결과를 통한 간접 확인도 시도하지 못했고, (2) WebFetch
직접조회는 위와 같이 hallucination이 강하게 의심되어 신뢰할 수 없었다.

[한계 및 후속 작업 제안]
- 실제 신뢰할 수 있는 게시글 목록을 얻으려면 (1) 브라우저 자동화 도구(headless
browser)로 JavaScript 렌더링 후 크롤링하거나, (2) WebSearch 예산이 복구된
세션에서 "sdm.go.kr 구청장에게 바란다 2026" 등의 키워드로 개별 매체·블로그에
인용된 민원 사례를 찾는 방식이 필요함
- "옴부즈만" 등 타 민원 채널의 존재 여부 자체도 이번 조사에서는 확인하지
못했음(WebSearch 예산 소진 이전에 조사 대상에 포함하지 못함) - 확인안됨으로
명시
- 후속 조사 시 이 파일의 "접근불가" 판정을 재검증하기 위해, 동일 URL을 3회 이상
반복 조회해 내용 일관성을 먼저 테스트하는 절차를 다른 자치구 조사에도 표준
검증 절차로 도입할 것을 제안함(이번 조사에서 우연히 발견한 방법론이나, 향후
WebFetch 기반 조사 전반의 신뢰성 검증에 유용함)
