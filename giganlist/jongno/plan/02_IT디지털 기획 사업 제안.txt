# IT/디지털 기획 사업 제안 (필수 포함 항목)

=================================================
[IT-1] 종로구 사업예산 투명공개 플랫폼 구축
=================================================
우선순위: 최우선
추정예산: 400~550백만원(1회성 구축비 280~380 + 연 운영비 60~85 × 2년)
제안부서(추정): 기획예산과 + 디지털행정과(구축), 전 부서(예산 데이터 제공)

[문제 정의]
spec/07 조사 결과, 2026년 종로구 5대 분야 33개 사업 중 예산액이 구체적으로 확인된
사업은 단 1건(한양도성 창의문 구간 복원사업, 46.8억원)뿐이며 나머지 32건(약 97%)은
"확인안됨"으로 분류됨. 원인은:
1. 2026년도 예산안(5,813억원) 보도자료가 "5대 분야별 대표사업 나열" 방식이라
사업별 배분액이 공개되지 않음
2. 유일한 예산 세부문서로 추정되는 종로구의회 위원회별 "2026년도 주요업무계획보고"
PDF가 이미지 임베딩 방식이라 검색엔진 색인이 되지 않고, 조사자도 로컬 OCR 도구
없이는 열람이 불가능함(spec/07 한계 사항)
3. 서울시 열린재정 포털(openfinance.seoul.go.kr/jongno)의 "사업별 예산정보"
조회 페이지는 실제 존재하나 부서명 등 특정 조건을 입력해야 데이터가 조회되는
구조로, 첫 화면에서는 "등록된 데이터가 없습니다"로 표시되어 일반 구민이 이용하기
어려움(본 조사에서 직접 접속 확인)

[제안 내용]
1. 종로구의회 회의록시스템(bookcouncil.jongno.go.kr)에 이미 존재하는 위원회별
"2026년도 주요업무계획보고" 자료를 원본 PDF 대신 구조화된 텍스트/표 데이터로
재입력(부서 협조 필요)하여 홈페이지에 공개
2. 서울시 열린재정 포털의 "사업별 예산정보" API를 종로구 홈페이지에 임베드하고,
부서명을 몰라도 사업명/키워드로 바로 검색되도록 기본 검색 UI를 구청 자체 페이지에
재구성(열린재정 포털 원본은 그대로 두되 종로구 접점을 새로 설계)
3. 5대 분야(도시환경/보육교육/문화관광/복지건강/안전스마트행정) 필터 및 사업별
상세페이지(사업명, 담당부서, 예산현액, 집행률, 진행상태) 표준 양식 마련
4. 2026년도 예산안 보도자료에 나열된 사업명(예: "청운별빛어린이집 건립", "종로
아트버스 운행" 등)을 실제 예산 항목과 1:1 매핑하는 마스터 코드 부여 - 언론보도
문구와 실제 예산과목명이 다른 경우가 많다는 점(spec/07에서 반복 확인된 패턴)을
해소하기 위함

[기술 스택 권장]
- 백엔드: 서울시 열린재정 포털 API 연동 우선(신규 개발 최소화)
- 프론트엔드: 반응형 웹, 검색엔진 색인 가능하도록 서버사이드 렌더링 적용
- 위원회 자료: 최소한 신규 발간분부터는 원본 한글(HWP)/텍스트 PDF 병행 제공을
의회사무국에 요청(과거분 전체 OCR 전환은 예산 상 후순위로 배치)

[기대효과]
- 33개 사업 중 예산 확인율 3%(1건) -> 목표 70% 이상으로 개선(위원회 자료 공개
전환 시점 기준)
- "얼마가 어디에 쓰였는지" 구민 문의에 단일 창구로 응답 가능
- 향후 민선9기 4개년계획 수립 시에도 예산-사업 매핑 마스터데이터를 그대로 활용 가능

[IT-2] 민원 통합관리 & 옴부즈만 접근성 개선 대시보드
=================================================
우선순위: 최우선
추정예산: 300~420백만원(구축비 240~320 + 연 운영비 30~50)
제안부서(추정): 감사담당관(옴부즈만 소관 추정) + 디지털행정과

[문제 정의]
spec/08 조사 결과:
1. 종로구의 실사용 민원 게시판은 "구청장에게 바란다"(공개 목록 열람 가능, 확인된
15건 전부 "답변완료")와 "옴부즈만"(제도 존재는 확인되나 2026년 실제 게시글 현황은
확인 불가, 로그인 필요 구조로 추정)으로 이원화되어 접근성 격차가 있음
2. "구청장에게 바란다"는 답변완료 전까지 비공개 처리되어, 접수했으나 아직 처리중인
민원이 얼마나 있는지, 평균 처리기간이 얼마인지 구민이 알 수 없는 구조(공개된 15건은
모두 결과가 나온 것만 보이는 "생존자 편향" 구조)
3. 15건의 담당부서가 보건정책과·재무과·도로과·교통행정과·가로정비과·청소행정과·
도시녹지과·도시개발과·평생교육과·관광체육과 등 10개 이상으로 분산되어 있어, 부서
간 유사 민원(예: 흡연구역 관련 962·957) 중복 처리 가능성이 있음

[제안 내용]
1. "구청장에게 바란다"와 옴부즈만 신청 건을 동일한 접수번호 체계로 통합 관리하는
백엔드 대시보드 구축 - 담당 공무원이 채널 구분 없이 단일 화면에서 처리
2. 구민용 "내 민원 조회" 페이지: 접수번호 기반으로 두 채널 모두 진행상황 확인
가능(접수 -> 배정 -> 검토중 -> 답변완료 공통 상태값), 처리중 건도 익명화된 통계로
"현재 처리중 몇 건" 정도는 공개해 불투명성 해소
3. 유사 민원 자동 태깅: 키워드(흡연구역, 보행신호등 등) 기반으로 유사 민원을
자동 그룹핑해 담당부서가 중복 대응을 피하고 정책화 우선순위를 판단할 수 있도록 지원
4. 옴부즈만 채널의 로그인 장벽을 낮추기 위해 "구청장에게 바란다"와 동일한 수준의
비회원 조회(접수번호+휴대폰 인증) 방식을 옴부즈만에도 적용 검토
5. 연 1~2회 "민원처리 통계 공표"(부서별 접수건수, 평균 처리일수 등) 자동 생성

[기대효과]
- 15건의 확인된 민원 중 창신23정비사업(spec/02, 5분야 spec/06 도시개발과 소관)처럼
정책 사각지대에서 발생한 민원을 조기에 포착해 정책화 가능
- 두 채널 간 접근성 격차 해소로 옴부즈만 이용률 제고 기대
- 향후 IT-1(예산공개플랫폼)과 연동해 "이 민원이 어느 예산 사업과 관련있는지"까지
연결 가능

[IT-3] 도시재정비·정비사업 진행상황 안내 시스템
=================================================
우선순위: 상
추정예산: 200~300백만원
제안부서(추정): 도시개발과 + 미래도시추진과

[문제 정의]
spec/02: 종로구는 신속통합기획 등과 연계해 30개 구역·1만9,479세대 규모의 대규모
도시재정비를 추진 중이나, 구역별 진행상황을 종합 안내하는 창구는 확인되지 않음.
spec/08: 실제 "구청장에게 바란다" 게시판에 "창신23정비사업 선거관리위원회 결정
관련 감사 요청"(게시글 954, 2026.4.24, 도시개발과 답변완료) 민원이 접수된 사실이
확인됨 - 이는 정비사업 진행 과정(조합 설립, 선거관리위원회 구성·결정 등)에 대한
정보 비대칭이 실제 민원·감사요청으로 이어진 구체적 사례임.

[제안 내용]
1. 30개 재정비 구역 전체를 지도 기반으로 시각화하고, 구역별 현재 단계(기본계획
수립/정비구역 지정/조합설립/사업시행인가/관리처분/착공 등)를 표준화된 단계값으로
표시하는 "종로 정비사업 진행현황 지도" 구축
2. 창신23정비사업처럼 조합 내부 의사결정(선거관리위원회 결정 등)과 관련된 민원이
반복될 경우를 대비해, 각 구역 페이지에 "조합 관련 민원/감사 신청 안내" 링크를
표준으로 배치
3. 수송 도시정비형 재개발구역 1-2지구(spec/02, 민간시행)처럼 구청사 이전과
직결되는 대형사업은 별도 상세페이지로 분리해 공정률·설계 변경사항을 주기적으로 갱신
4. IT-2(민원 대시보드)와 연동해 구역별 민원 발생 현황을 담당부서가 모니터링

[기대효과]
- 정비사업 관련 정보 비대칭 해소로 창신23 유형의 감사요청성 민원 감소 기대
- 대규모 도시재정비 사업(19,479세대 영향권)에 대한 주민 신뢰도 제고

[IT-4] 민선9기 공약이행 추적 플랫폼
=================================================
우선순위: 상
추정예산: 150~250백만원
제안부서(추정): 기획예산과 + 홍보과

[문제 정의]
spec/06: 신임 유찬종 구청장(민선9기, 2026.6 당선·7월 취임)의 6대 구정비전 중
5개 세부공약(AI안심통학로 조성, 도로열선 확대, 구청장직속TF팀 신설, 찾아가는
구청장실, 24시기동대응센터)이 조사 시점(취임 약 1개월) 기준 예산·세부계획이
확정되지 않은 "선언" 단계임(spec/07에서 전부 "부분확인"으로 분류). spec/01에서
확인된 대로 종로구는 통상 취임 후 100일 이내에 공약을 확정하고 이후 4개년
실천계획을 수립하는 절차를 거치므로, 지금이 바로 이 계획 수립 과정에 투명성
설계를 반영할 수 있는 시점임.

[제안 내용]
1. 타 자치구(예: 도봉구·광진구 등)의 "공약 로드맵/공약 가계부/공약 지도" 형식을
참고해, 종로구 민선9기 공약이 확정되는 즉시 이행 단계(공약확정 -> 실천계획수립 ->
예산반영 -> 착수 -> 완료)를 구민이 추적할 수 있는 페이지를 사전 설계
2. 5개 세부공약을 포함한 전체 공약에 대해, IT-1(예산공개플랫폼)과 동일한 코드
체계로 예산이 반영되는 순간부터 자동 연동되도록 설계
3. "찾아가는 구청장실"(공약 중 하나)에서 수집되는 현장 의견을 IT-2 민원
대시보드와 연동해 별도 채널로 분류·집계
4. 100일 공약 확정 시점(취임 후 약 100일, 조사 시점 기준 약 2개월 후 예상)에 맞춰
1차 공개 목표로 로드맵 수립

[기대효과]
- 신규 행정부 출범 초기부터 투명성을 내장해 향후 4개년계획 수립 시 "예산 따로,
공약 따로" 식의 문제 재발 방지
- 취임 초기 관심도가 높은 시점을 활용해 구민 참여도 제고
