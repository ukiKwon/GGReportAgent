IT/디지털 기획 사업 제안 (필수 포함 항목)

[IT-1] 신규사업 개시 알림형 통합 사업정보 페이지 구축
=================================================
우선순위: 최우선
추정예산: 350~500백만원 (구축비 250~350 + 연 운영비 50~75 × 2년)
제안부서(추정): 기획조정과(주요업무계획 발간 부서) + 정보통신팀, 전 부서(콘텐츠 제공)

[문제 정의]
spec/07 조사 결과, 대표성 있는 약 22개 사업 검색조사에서 확인안됨(9%)·부분확인(32%)으로
분류된 사업 대부분이 "2026년 신규" 사업이었음(효도카드 출시, 동작청년센터 개관, 어린이
실내놀이터 JUMP1, 서달산 스카이워크 전망대, 동작 반려견 공원, 동작탄소중립 마일리지,
9988 동작체력인증센터). spec/07의 결론을 그대로 인용하면: "2026년 '신규' 사업은 문서
발행시점(2026년2월)과 조사시점(2026년7월) 사이 간극이 있음에도 불구하고 아직 별도
홍보페이지가 구축되지 않은 경우가 많아 확인율이 낮은 편"임. 특히 "JUMP1"처럼 고유
브랜드명을 붙인 사업조차 검색이 전혀 안 되는 사례(확인안됨)가 존재.

[제안 내용]
1. "2026년 동작구 주요업무계획"(95개 중점사업)을 발간 시점(매년 2월)에 맞춰 마스터 DB화
   하고, 각 사업별로 "예정(계획 발표)" -> "준비중" -> "개시(개관/시행)" -> "운영중" 4단계
   상태값을 부여해 실시간 갱신하는 단일 페이지 신설
2. 특히 "신규 개소·개관형" 사업(효도카드, 동작청년센터, JUMP1, 스카이워크 전망대 등
   구체적 개관 일정이 있는 사업)은 개소 D-30일 전부터 자동으로 안내 배너 노출 -
   "완공 후 정보 공백기"를 사전에 없애는 방식
3. 세대별 6개 챕터 + 분야별 7개 챕터 분류 체계를 그대로 검색 필터로 사용(주민이 이미
   책자에서 익숙한 분류 유지), 검색엔진 색인이 가능하도록 정적 렌더링 페이지로 구축
4. 청년정책 전용사이트(youthjob.dongjak.go.kr)처럼 신설 전용 사이트를 또 만들기보다,
   기존 메인 포털(www.dongjak.go.kr) 내 페이지로 통합해 유지보수 주체를 일원화

[기술 스택 권장]
- 검색엔진: 경량 오픈소스(Meilisearch 등)
- 프론트엔드: 정적 사이트 생성(SSG) 방식으로 검색엔진 색인 최적화
- 백엔드: 기획조정과의 연간 계획 수립 프로세스(매년 2월 발간)에 맞춰 신규사업 리스트를
  연 1회 일괄 등록 + 개별 부서가 상태값만 갱신하는 저부담 구조

[기대효과]
- spec/07 확인안됨·부분확인 사업 비율(41%) -> 목표 15% 이하로 개선
- 특히 신규 개관형 사업의 "개관 직후 정보 공백기"를 구조적으로 제거
- 매년 갱신되는 주요업무계획의 특성(도봉구 4개년계획과 달리 단년도 계획)에 맞춰
  연 1회 마스터 데이터 갱신만으로 유지보수 가능

[IT-2] 민원채널 접근성 복구 & 처리현황 공개 대시보드
=================================================
우선순위: 최우선
추정예산: 300~450백만원 (구축비 250~350 + 연 운영비 50~100)
제안부서(추정): 감사담당관(옴부즈만 소관) + 정보통신팀

[문제 정의]
spec/08 조사 결과, "구청장에게 바란다"와 "옴부즈만 민원 신청" 게시판은 WebFetch로
2회 이상 접근을 시도했으나 모두 "빈 페이지"로 반환됨 - 즉 외부에서 게시글 존재 여부,
제목, 처리상태를 전혀 확인할 수 없는 완전 차단 상태(JavaScript SPA 렌더링 방식으로 추정).
그 결과 spec/08 요약에 명시된 대로 "2026년 동작구 최다 민원 유형을 도봉구 수준으로
구체적으로 도출하지 못함" - 이는 분석 도구의 한계일 뿐 아니라, 일반 주민이나 검색엔진도
동일한 접근 장벽을 겪는다는 의미. 또한 "국민신문고 직접 링크"는 확인안됨 상태로,
민원상담 안내페이지에 "국민신문고와 연계"한다는 서술만 있고 실제 연결 경로가 불명확함.

[제안 내용]
1. "구청장에게 바란다"·"옴부즈만 민원 신청" 게시판을 검색엔진이 색인 가능한 서버사이드
   렌더링(SSR) 또는 정적 목록 페이지로 병행 제공(기존 SPA 상세조회 기능은 유지하되,
   목록/제목/접수일/처리상태만이라도 정적 페이지로 노출)
2. 옴부즈만 자료실이 이미 안정적으로 운영 중인 "연1회 성과보고서 게시 패턴"(spec/08,
   2019~2025년 7개년 자료 확인됨)을 벤치마크로 삼아, "구청장에게 바란다"에도 분기별
   처리현황 요약(접수/처리중/완료 건수, 평균 처리일수)을 별도 정적 페이지로 공표
3. 주민용 "내 민원 조회": 접수번호+휴대폰 인증으로 진행상황 확인, 카카오 알림톡으로
   상태 변경(답변완료 등) 자동 통보
4. 메인 포털 하단(푸터)에 국민신문고 직접 링크를 명시적으로 노출해 spec/08에서 지적된
   "연계 서술만 있고 링크 미확인" 상태를 해소

[기술 스택 권장]
- 기존 새올전자민원창구 시스템과의 연동 어댑터 개발(전면 교체보다 목록 데이터만
  별도 API로 뽑아 정적 페이지 생성)
- 알림톡: 카카오 비즈메시지 API

[기대효과]
- 외부 분석·검색·주민 접근이 모두 막혀 있던 핵심 민원채널 2곳의 최소한의 투명성 확보
- 처리현황이 최초로 정량 데이터화되어, 이후 정책 수립 시 실제 민원 패턴 기반 대응 가능
- 국민신문고 연계 경로 명확화로 대외 평가지표(spec/01: 민원서비스종합평가 우수-행정
  안전부/국민권익위원회 수상 이력)의 지속 유지에도 기여

[IT-3] 동작구의회 회의록 연계 사업추적 검색 위젯
=================================================
우선순위: 상
추정예산: 120~180백만원
제안부서(추정): 기획조정과 + 의회사무국

[문제 정의]
spec/07: 검색 결과 다수가 동작구 자체 홈페이지가 아닌 동작구의회(assembly.dongjak.go.kr)
회의록 페이지로 나타남. 노량진 옛청사부지 복합개발(2025.8 실시협약 변경, 2025.9 사업협약
체결), 신속통합기획 상도14·15구역·사당4동(후보지선정~가이드라인 확정 단계), 이수-과천
복합터널(2023.12 서울시-롯데건설 협약 후 착공 순연), 구민안전보험 화상수술비 증액
(30만원->100만원) 등 주요 사업의 구체적 진행경과와 수치가 모두 의회 회의록을 통해서만
확인됨. 도봉구의 "majorWork DB" 같은 구 자체 통합 추적 페이지가 부재.

[제안 내용]
1. 의회사무국이 이미 보유한 회의록 검색 시스템(assembly.dongjak.go.kr)의 API 또는
   크롤링 연동으로, IT-1 통합 사업정보 페이지 내 각 사업 항목에 "관련 의회 회의록"
   링크를 자동 매칭해 노출하는 위젯 개발
2. 특히 대형 개발사업(신속통합기획 16개소, 모아타운 5개소, 재정비촉진구역 등 spec/03)은
   의회 상임위원회별(복지건설위원회, 행정재무위원회, 예산결산특별위원회) 회의록 언급
   빈도가 높으므로 이를 우선 연동
3. 키워드 매칭 방식: 사업명 + 관련 구역명(예: "상도14구역", "이수-과천")을 색인 키워드로
   등록해 회의록 검색과 사업정보 페이지 간 상호 링크 구성

[기대효과]
- 주민이 "구청 홈페이지에는 없고 의회 회의록에만 있는" 정보 격차를 스스로 찾아다니지
  않아도 되도록 자동 연결
- 별도의 신규 콘텐츠 생산 없이 기존 공개된 의회 회의록을 재활용하는 저비용 방식

[IT-4] 세대별 생애주기 사업 안내 챗봇/카카오톡 알림
=================================================
우선순위: 중
추정예산: 100~150백만원
제안부서(추정): 청년청소년과 등 세대별 소관부서 + 정보통신팀

[문제 정의]
spec/02: 세대별 6개 챕터(어르신9+청년7+청소년6+어린이7+영유아5+임신출산5=39개
중점사업)는 "개별 사업 예산액이 대부분 카드형 설명에 명시되지 않음(도봉구 문서와 달리
연차별 투자계획표가 세대별 챕터에는 없음)" - 즉 사업은 다양하게 존재하나 주민이 자신의
생애주기 단계에서 어떤 지원을 받을 수 있는지 한눈에 파악하기 어려운 구조.

[제안 내용]
1. 주민이 본인 생애주기 단계(예: 임신, 만0~5세 자녀, 청소년 자녀, 청년, 만65세 이상 등)를
   선택하면 해당 단계의 2026년 중점사업(효도카드/청년 문화생활비/수학여행경비지원/
   1동1키즈카페/HPV검사비 등)을 카카오톡 채널로 자동 안내받는 알림 서비스
2. 이미 확인됨으로 분류된 사업(청년·신혼부부 전월세 지원 - 2026.2.2~2.27 신청기간
   실제 확인, spec/07)의 신청기간 임박 알림을 우선 시범 적용
3. IT-1 마스터 DB를 데이터 소스로 재사용해 별도 콘텐츠 제작 부담 최소화

[기대효과]
- 예산 미기재로 인지도가 낮은 생활밀착형 사업의 체감도 제고
- 청년정책 전용사이트(youthjob.dongjak.go.kr)에서 검증된 "신청공고 명확 안내" 방식을
  세대별 전 분야로 확산
