# IT/디지털 기획 사업 제안 (필수 포함 항목)

=================================================
[IT-1] 재개발·정비사업 통합 안내 플랫폼
=================================================
우선순위: 최우선
추정예산: 250~350백만원(구축비 200~280 + 연 운영비 25~35 × 2년)
제안부서(추정): 도시재생과 + 스마트정책과(구축), 관련 사업부서(데이터 제공)

[문제 정의]
spec/02 조사 결과, 마포구는 아현1구역 공공재개발(3,476가구), 염리동488-14
재개발(1,120세대), 상암DMC 랜드마크용지 7차공급, 서부운전면허시험장 부지
전환 등 대형 재개발·개발 사업 7건이 동시에 진행 중이나, 각 사업의 진행상황을
종합 안내하는 창구가 확인되지 않음. 구민이 자신이 속한 구역의 진행단계(정비구역
지정/조합설립/사업시행인가 등)를 확인하려면 서울시 도시계획위원회 회의록, 개별
언론보도 등을 일일이 찾아봐야 하는 구조임.

[제안 내용]
1. 마포구 내 대형 재개발·정비·개발 사업(7건 이상)을 지도 기반으로 시각화하고,
구역별 현재 단계를 표준화된 단계값(기본계획수립/정비구역지정/조합설립/사업시행
인가/관리처분/착공/준공)으로 표시
2. 서울시 도시계획위원회 의결 결과(정비구역 지정, 지구단위계획 등)를 정기적으로
크롤링/수동 갱신하여 최신 진행상황을 반영
3. 상암DMC 랜드마크용지처럼 매각 공고 방식의 사업은 별도 "공고/입찰 정보" 섹션을
마련해 매각 조건 변경 이력(6차 실패->7차 조건완화 등)을 투명하게 기록
4. 각 구역 페이지에 관련 민원·문의 안내 링크(마포옴부즈만 등)를 표준 배치

[기대효과]
- 재개발 정보 비대칭 해소로 주민 신뢰도 제고
- 7건 이상의 대형 사업에 대한 문의를 단일 창구로 응대 가능
- 향후 신규 재개발 사업 추가 시에도 동일 플랫폼으로 확장 가능

[IT-2] 사업예산 구조화 공개 플랫폼
=================================================
우선순위: 최우선
추정예산: 350~480백만원(구축비 280~380 + 연 운영비 35~50 × 2년)
제안부서(추정): 기획예산과 + 스마트정책과(구축), 전 부서(예산 데이터 제공)

[문제 정의]
spec/01 조사 결과, 마포구 홈페이지에 실재하는 "2026년 주요업무계획"(156페이지
PDF)을 실제로 다운로드하여 pdftotext로 텍스트 추출을 시도한 결과, 본문에 사용된
폰트가 ToUnicode CMap이 없는 서브셋 폰트로 저장되어 있어 한글 사업명·설명이
전부 공백으로 치환됨을 확인함. 반면 예산으로 추정되는 숫자(13,700/1,360/
41,552/12,700/109,795/3,180/90,702/410,864 등, 단위 불명)와 날짜, 목차
구조(5대분류x3소분류)는 추출에 성공함 - 즉 "문서 안에 개별 사업 예산 수치
자체는 존재하지만, 어떤 사업의 예산인지는 구민도 조사자도 알 수 없는" 구조임.
이는 종로구의 이미지 임베딩 PDF 문제와 원인은 다르지만 결과(비접근성)는 동일함.

[제안 내용]
1. "2026년 주요업무계획" 원본 한글(HWP) 파일을 발간부서(기획예산과 추정)로부터
확보하여, PDF 변환 시 반드시 텍스트 레이어에 ToUnicode CMap이 포함된 표준
폰트로 재변환(현재 서브셋 폰트 임베딩 방식 대신)하는 내부 발간 가이드라인 수립
2. 최소한 신규 발간분(2027년도 주요업무계획)부터는 이 표준을 적용하고, 과거
발간분(2026년도 등)은 원본 HWP 파일 자체를 병행 공개
3. 마포구 홈페이지 "열린행정 > 열린재정" 메뉴와 연동해 사업명-예산과목-부서를
1:1 매핑하는 마스터 코드 부여
4. 5대 분야 필터 및 사업별 상세페이지(사업명, 담당부서, 예산현액, 집행률,
진행상태) 표준 양식 마련

[기술 스택 권장]
- 백엔드: 서울시 열린재정 관련 데이터 연동 우선 검토(신규 개발 최소화)
- 문서 표준화: HWP -> PDF/A 변환 시 텍스트 레이어 표준 준수 여부를 발간 전
자동 검증하는 스크립트 도입(예: pdftotext 결과에 한글 문자 비율이 일정 기준
이하면 발간 반려)
- 프론트엔드: 반응형 웹, 검색엔진 색인 가능하도록 서버사이드 렌더링 적용

[기대효과]
- "2026년 주요업무계획"이 사실상 읽을 수 없었던 문제(spec/01)를 해소해 향후
발간분부터는 실질적 열람 가능
- "얼마가 어디에 쓰였는지" 구민 문의에 단일 창구로 응답 가능

[IT-3] 민원채널 통합 대시보드(마포옴부즈만+국민신문고 연동)
=================================================
우선순위: 상
추정예산: 250~350백만원(구축비 200~280 + 연 운영비 25~35 × 2년)
제안부서(추정): 감사담당관 + 스마트정책과

[문제 정의]
spec/08 조사 결과:
1. "구민에게 듣겠습니다" 게시판이 2025년 12월 17일부터 국민신문고로 연계
전환되면서, 마포구 자체 홈페이지에서는 2026년 이후 일반 민원 현황(접수건수,
유형 등)을 더 이상 확인할 수 없게 됨
2. 반면 "마포옴부즈만" 고충민원 게시판은 로그인 없이 목록 열람이 가능해,
2026년 실제 게시글 27건(순번 760~786)이 확인됨. 즉 두 채널의 접근성이
비대칭적으로 뒤바뀐 상태(한쪽은 이관으로 단절, 한쪽은 그대로 공개)
3. 27건 중 "삭제된 민원글입니다"로 표시된 게시글이 3건 있으나 삭제 사유는
공개되지 않아 구민이 왜 삭제되었는지 알 수 없는 구조

[제안 내용]
1. 국민신문고로 이관된 일반 민원 중 마포구 소관 처리 현황(접수건수, 유형별
통계 등 익명화된 요약 통계)을 국민신문고 공공데이터/API를 통해 다시 마포구
홈페이지에 요약 형태로 게시하는 연동 체계 구축(개별 민원 원문이 아닌 통계
수준으로)
2. 마포옴부즈만 게시판에 "유사 민원 자동 태깅"(쓰레기·담배 등 생활환경 민원이
27건 중 9건으로 최다 - spec/08) 기능을 추가해 반복 민원의 정책화 우선순위
판단을 지원
3. 삭제된 게시글에 대해 "삭제 사유"를 최소한의 범주(작성자 요청/규정 위반 등)로
구분해 표시하는 방식 도입 검토
4. 향후 IT-2(예산공개플랫폼)와 연동해 "이 민원이 어느 예산 사업과 관련있는지"
연결 가능하도록 설계

[기대효과]
- 국민신문고 이관으로 발생한 마포구 자체 민원 통계 공백 해소
- 옴부즈만 채널의 반복 민원(생활환경 33%)을 조기 포착해 정책화 가능
- 삭제 사유 표시로 구민의 신뢰도 제고

[IT-4] 민선9기 공약이행 추적 플랫폼
=================================================
우선순위: 상
추정예산: 150~250백만원
제안부서(추정): 기획예산과 + 홍보과

[문제 정의]
spec/04, spec/01: 신임 유동균 구청장(민선9기, 2026.6.3 당선·7.7 취임)의 핵심
공약인 "공덕-홍대-합정-망원-한강 문화벨트 조성"과 "재개발·재건축 신속추진
TF팀 구성"이 조사 시점(취임 약 2주) 기준 방향성 선언 수준이며(spec/07에서
"부분확인"으로 분류), 구체 사업·예산은 확정되지 않은 상태임.

[제안 내용]
1. 문화벨트 조성, TF팀 신설 등 주요 공약이 확정되는 즉시 이행 단계(공약확정 ->
실천계획수립 -> 예산반영 -> 착수 -> 완료)를 구민이 추적할 수 있는 페이지를
사전 설계
2. IT-1(재개발 안내 플랫폼)과 연동해, "재개발·재건축 신속추진 TF팀"이 실제
가동되면 IT-1의 개별 구역 페이지에 TF팀 개입 여부를 표시하는 구조로 설계
3. IT-2(예산공개플랫폼)와 동일한 코드 체계로 공약 예산이 반영되는 순간부터
자동 연동되도록 설계

[기대효과]
- 신규 행정부 출범 초기부터 투명성을 내장해 향후 임기 중 "공약 따로, 예산
따로" 문제 예방
- 취임 초기 관심도가 높은 시점을 활용해 구민 참여도 제고
