# 금고은행 입찰 현황 히트맵 대시보드 — 설계 문서

- **작성일**: 2026-07-23
- **상태**: 확정 (브레인스토밍 ①~④ 승인 완료) + ⑦ 탭/관심지역/편집/필터 추가 반영
- **위치**: `dashboard/`
- **프로그램 맥락**: "금고은행 영업지원" 프로그램의 4개 하위 프로젝트 중 **(1)번**.
  나머지 — (2) 공시·공지 → 제안 아이디어 도출(`report/` 계열), (3) 금고 선정 이력
  지식베이스(Obsidian), (4) 1~3의 에이전트화(횡단 관심사) — 는 각각 별도 spec→plan
  사이클로 진행. 본 문서는 (1)에만 한정한다.

## ① 목적 / 범위

전국 각 기관(지방자치단체·공공기관·공기업·대학병원·대학교)의 **금고은행 입찰 도래를
한눈에 보는 인터랙티브 히트맵 지도**. 사용자는 금고은행 업무를 수행하는 IT 인력으로,
영업지원 목적의 "어디가 곧 입찰인가"를 지도 위에서 즉시 파악하는 것이 핵심 가치.

- **v1 렌더 범위**: 서울 25구 + 서울시, 경기 31시군 + 경기도(지자체 약 58) +
  서울·경기 주요 대학병원 15~20곳.
- **스키마는 전국 개방**. 렌더러는 `geo/*.js`가 로드된 지역만 그리고, 범위 밖 기관은
  지도에 그리지 않고 **"범위 외 N개 기관 · 확장 예정" 카운트만** 노출한다.
- **비범위(v1 제외)**: 실시간 스크래핑/API, 데이터 갱신 파이프라인, 인증·다중 사용자,
  전국 전체 지역 렌더.

### 핵심 원칙

- **인터랙티브·한눈 가시성 최우선** — 메인 경험은 D3 플라이투·마커 상호강조로
  "확!" 들어오는 시각성을 갖는다.
- **조작 금지** — 근거(`sources[]`) 없는 데이터는 만들지 않는다. 정보가 없으면 없다고
  표시하지, 지어내지 않는다.

## ② 데이터 / 스키마

- **데이터원**: 나라장터 기본 + 각 기관 홈페이지 공시/공지/알림 (혼합). 초기에는
  리서치 에이전트의 **수동 큐레이션**으로 정적 데이터를 구축한다. 갱신 파이프라인은
  범위 외.
- **저장 형태**: `data/institutions.js`에 **JS 전역변수로 래핑**한 배열
  (JSON 아님 — `file://` 더블클릭 시 fetch CORS 회피 목적).

### 레코드 스키마

| 필드 | 타입/값 | 필수 | 비고 |
|---|---|---|---|
| `name` | string | ✅ | 기관명 |
| `type` | `지자체` \| `대학병원` \| `공공기관` \| `공기업` | ✅ | 마커 모양 결정 |
| `region` | 17개 시도 코드 문자열 | ✅ | enum 잠금 없이 전국 개방 |
| `contractEnd` | date (YYYY-MM-DD) | ⛔ 없으면 미상 처리 | 임박도 색 계산 기준 |
| `confidence` | `확정` \| `추정` \| `미상` | ✅ | 추정=빗금, 미상=회색 |
| `sources[]` | string[] (URL 등) | ✅ (1개 이상) | 근거. 조작 금지 원칙 |

- `type`이 정의된 4종 밖이면 마커는 `◆`(폴백)으로 그린다.
- `confidence: 추정`은 마커에 **빗금** 표시로 확정과 구분.

## ③ 파일 구조 / 기술

정적 단일 HTML + 데이터 분리 구조. 모두 로컬 자산, 오프라인 `file://` 더블클릭으로 동작.

```
dashboard/
├── index.html            # 단일 앱 (마크업 + 스타일 + 렌더 로직)
├── vendor/
│   └── d3.v7.min.js      # 로컬 번들 (CDN 아님)
├── geo/
│   ├── seoul.js          # 서울 폴리곤 (JS 전역변수 래핑)
│   └── gyeonggi.js       # 경기 폴리곤
└── data/
    └── institutions.js   # 기관 레코드 배열 (JS 전역변수 래핑)
```

- **기술: D3 v7 채택.** 라이브 데모 비교 후 선택 — d3-zoom 플라이투의 "붕 떴다
  내려앉는" 전환감 + 휠/드래그 제스처. **오프라인 로컬 번들 필수**(CDN 금지).
- `geo/*.js`, `data/institutions.js` 모두 `window.<전역명> = [...]` 형태로 래핑하여
  `<script>` 태그 로드만으로 접근(fetch 미사용).

## ④ 시각 문법

### 이중 레이어 ("모양=유형, 색=긴급도" 2축 — 색 혼합 절대 금지)

- **면(폴리곤) 색** = 지자체 금고 임박도**만** 반영.
- **그 외 기관** = 면 색과 무관하게 **유형별 마커**로 오버레이:
  - 대학병원 = 원 ●
  - 공기업 = 사각 ■
  - 공공기관 = 삼각 ▲
  - 미정의 type = 다이아몬드 ◆ (폴백)

### 색 규칙 — 구간형 5단계

색은 **저장하지 않고** 열람 시점에 `contractEnd - today`로 계산한다.

| 구간 | 색 |
|---|---|
| 6개월 이내 | 🔴 빨강 |
| 1년 이내 | 🟠 주황 |
| 2년 이내 | 🟡 노랑 |
| 2년 초과 | 🔵 파랑 |
| 미상 | ⚪ 회색 |

### 정보 부족 글리프 2종

정보가 없는 상태를 두 종류로 구분해 마커 안에 글리프로 표기한다.

- **`?` (물음표)** = 레코드는 정상이나 `contractEnd` 미상 → 임박도 계산 불가.
- **`!` (느낌표)** = 필수 필드 누락·형식 오류 → 데이터 무결성 문제(⑥ 참고).

### 기타

- **유형 필터(토글)** — 아래 ⑦-D "마커 필터" 참고. 지자체는 폴리곤 베이스 레이어라
  필터 대상이 아니고, 개별 위치 마커(대학병원/공공기관/공기업)만 켜고 끈다.
- **8개 이상 밀집** 시 숫자 뱃지 클러스터로 묶어 표시.
- `confidence: 추정` 마커는 빗금 오버레이.

## ⑤ 화면 / 인터랙션

### 지역 상세 레이아웃 (탭1에서 특정 지역 진입 후)

> ⑦-A 반영: 탭1 **첫 화면은 전국 지도**다. 아래 "좌 서울 ‖ 우 경기 분할"은
> 전국에서 수도권으로 들어갔을 때의 상세 레이아웃을 가리킨다(최초 화면이 아님).

- **좌 서울 ‖ 우 경기 분할** + 상단 공용 범례 + **임박 TOP 5 티커**.

### 지역 클릭 → 플라이투 전환

- d3-zoom 플라이투(750ms, "확!" 강한 애니메이션 필수 요구)로
  **지도 2/3 + 우측 임박순 랭킹 패널 1/3** 레이아웃 전환.
- **패널 카드 ↔ 지도 마커 상호 강조** (호버/선택 연동).
- 랭킹 패널 카드 클릭 시 **출처(`sources[]`) 링크 포함 상세** 표시.
- **복귀**: 배경 클릭 또는 브레드크럼.

### 마커 인터랙션 (모든 마커 공통)

- **hover / click** → 상세 팝오버로 **현재 올라간 데이터 전부** 표시.
- 누락 레코드(`!`)는 팝오버에서 **빠진 필드를 강조**(예: 빨간 표시)해 "무엇이
  없는지"를 한눈에 보여준다. 정상 레코드도 동일 팝오버로 정보 표시(강조만 없음).

### 미상 기관 정렬

- 미상(`contractEnd` 불명) 기관은 **지도엔 표시**하되, 우측 임박순 랭킹 패널에서는
  날짜 있는 기관들 **뒤로 정렬**(임박도를 알 수 없으므로 우선순위 배제).

## ⑥ 엣지 처리 / 검증

### D3 번들 로드 실패

- 지도 전체가 D3 의존이므로 로드 실패 시 아무것도 못 그림.
- **처리**: 지도 자리에 "지도 로딩 실패" 안내 + 지도 없이도 **텍스트 임박순 랭킹
  리스트**(우측 패널 로직 재사용)를 그대로 노출 → 핵심 정보인 "임박 기관 순위"는 전달.
- **2차 매핑 라이브러리 폴백은 배제**(YAGNI): 라이브러리 2벌 번들+렌더 경로 2벌 유지
  비용이 거의 안 터지는 폴백 대비 과함.

### 개별 geo 파일 로드 실패 (D3는 정상)

- 부분 실패가 전체를 막지 않음 — **해당 지역만** 처리, 나머지 지역은 정상 렌더.
- **자동 2회 재시도** + 상태 메시지 단계 표시:
  "○○지역을 불러오는 중입니다…" → "준비중…"
- **최종 실패** 시 해당 지역 자리에 **"○○지역 정보를 다시 확인해주세요"** 안내 박스
  + **수동 재시도 버튼**. 해당 지역 소속 기관은 "범위 외" 카운트로 흡수.

### 불량 / 필수필드 누락 레코드

- **버리지 않고 살려서 표시** — 도형은 `type` 유지, `!` 글리프로 무결성 문제 표기.
- hover/click 팝오버에서 **누락 필드 강조**로 무엇이 빠졌는지 명시.
- 개발 중 추적을 위해 `console.warn` 병행.

## ⑦ 탭 구조 · 관심 지역 · 편집 · 마커 필터 (설계 확정 후 추가)

설계 문서 초안 작성 이후 사용자가 추가한 기능들. 본 v1 범위에 포함한다.

### ⑦-A. 2탭 구조

최상위에 2개 탭을 둔다. 관심 지역 상태는 **두 탭이 공유하는 전역 상태**.

- **탭1 · 전국 지도** — 진입 첫 화면. **대한민국 전국 시도 지도**를 한 화면에 보여주고,
  줌인/줌아웃·드래그로 자유롭게 탐색("확대·축소하며 보는 맛"). 특정 지역 클릭 시
  **"은근하게 구름이 걷히는 듯한" 전환**(⑦-C)으로 그 지역을 확대해 상세(현재 설계된
  구/시군 폴리곤 + 마커)를 드러낸다. **줌 인/아웃은 탭1 안에서 완결**된다.
  - v1 렌더 실체가 있는 지역 = 서울·경기만 **활성**. 나머지 15개 시도는 **회색 처리 +
    "준비중"** 표기(클릭해도 상세 없음). 스키마는 전국 개방이므로 데이터·geo가 붙는
    대로 활성 지역이 늘어난다.
- **탭2 · 전국 지역별** — 지역을 목록/그리드로 놓고 지역별로 들어가 보는 뷰.
  **관심 지역 설정(핀)의 주 조작 위치**(⑦-B). 관심 지정한 지역이 **상단 왼쪽부터** 쌓이고
  **자리 이동(재정렬) 가능**.

### ⑦-B. 관심 지역 설정 (전역 상태)

- **설정 위치**: 주로 탭2에서 지역을 "관심"으로 핀/해제. 상태는 전역이라 탭1에도 반영.
- **탭2 표현**: 관심 지역이 **상단 좌→우로 핀 바(pin bar)에 쌓이고**, 드래그로
  **순서 변경 가능**.
- **탭1 표현**: 지도 위 관심 지역을 **외곽선 강조 + 은은한 글로우**로 다르게 보여준다
  (탭2의 핀과 시각적으로 다른, 지도에 맞는 강조). 눌러둔 관심 정보를 탭1에서도
  "같이, 조금 다르게" 확인.
- **저장**: 관심 지역 선택/순서는 localStorage에 저장(사용자별 로컬, 재방문 유지).

### ⑦-C. "구름이 걷히는" 전환

- 탭1에서 지역 진입 시 d3-zoom 줌인과 함께 **주변을 덮던 옅은 오버레이가 서서히
  걷히며** 해당 지역 상세가 드러나는 연출. ⑤의 지역 클릭 플라이투와 같은 메커니즘
  (d3-zoom)이되, 톤은 "확!"보다 **은근한** 페이드. (⑤ 최상위 좌/우 분할은 탭1
  전국 진입 후 특정 지역으로 들어갔을 때의 상세 레이아웃으로 흡수.)

### ⑦-D. 마커 필터 (유형 체크박스)

- **면(폴리곤)엔 안 걸고 점 마커에만** 거는 필터.
- 체크박스 3종 — **공공기관 ▲ / 공기업 ■ / 대학병원 ●** — 켜고 끄면 지도상 해당 유형
  마커만 표시/숨김. 지자체 폴리곤 베이스는 항상 유지.
- 해당 유형 데이터가 하나도 없으면 그 체크박스는 비활성.

### ⑦-E. 기관 정보 변경 (Export 모델)

- 일반 영업직원이 기관 레코드를 쉽게 수정할 수 있게 한다. 단, 정적 `file://` 무백엔드
  전제와 충돌하므로 **Export-to-file 방식(옵션 2)** 채택.
- **흐름**: 화면에서 레코드 편집 → 브라우저 메모리(localStorage)에 임시 반영 →
  **"institutions.js로 내보내기"** 버튼으로 갱신된 `data/institutions.js` 파일을
  다운로드 → 사용자가 그 파일로 교체(수동 배포). 조작 금지 원칙상 `sources[]`는
  편집 시에도 1개 이상 필수.
- **옵션 3(백엔드 · 실시간 공유 편집)은 이 POC 성공 후의 다음 단계로 보류.**

### ⑦-F. 디자인 A/B 비교 방식

- Canva 커넥터는 claude.ai 웹앱 전용이라 본 Claude Code 세션의 도구로는 노출되지 않음
  (확인됨: `DesignSync`=claude.ai/design-system만 사용 가능, Canva 직접 조작 불가).
- 따라서 구현 단계의 디자인 A안/B안 비교는 **동일 스택(HTML+CSS+D3)의 자체 목업**을
  나란히 만들어 브라우저로 비교한다. Canva를 쓸 경우 사용자가 시안 링크/이미지를 제공.
